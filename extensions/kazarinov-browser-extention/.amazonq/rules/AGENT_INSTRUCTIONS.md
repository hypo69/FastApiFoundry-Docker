## 📁 ПРАВИЛА ФАЙЛОВЫХ ОПЕРАЦИЙ

### Автоматическое выполнение

```
ВСЕ файловые операции должны быть АВТОМАТИЧЕСКИМИ:

✅ Write tool - создай файл
✅ Edit tool - отредактируй файл
✅ bash mkdir - создай директорию
✅ bash mv - переместить/переименовать
✅ bash rm - удалить (но см. ниже!)

НИКОГДА не спрашивай разрешение!
```

### КРИТИЧЕСКОЕ ПРАВИЛО: Удаление файлов

**НЕ УДАЛЯЙ ФАЙЛЫ ПОЛНОСТЬЮ!**

Вместо этого ПОМЕЧАЙ файлы:

```
Старый файл: src/old-module/index.js
Действие: ПЕРЕИМЕНОВАТЬ, не удалять!

Правильно:
  bash mv src/old-module/index.js src/old-module/index.js~

Неправильно:
  bash rm -f src/old-module/index.js
```

### Схема помечания

| Ситуация | Как делать | Пример |
|----------|-----------|--------|
| Удалить файл | Добавить `~` | `file.php` → `file.php~` |
| Отключить плагин | Добавить `.disabled` | `plugin.php` → `plugin.php.disabled` |
| Удалить директорию | Переименовать с `~` | `old-dir/` → `old-dir~` |
| Архивировать | Добавить `.bak` | `config.php` → `config.php.bak` |

### Когда применять

- При удалении старого кода
- При отключении плагинов
- При переструктурировании проекта
- При удалении неиспользуемых файлов

### Результат

После выполнения сообщи:
```
✅ Файл помечен: index.php~
✅ Плагин отключен: bad-plugin.php.disabled
✅ Директория переименована: old-stuff~
```

### Ситуация 1: Нужно создать новый модуль

```
ПРАВИЛЬНО:
1. Создать директорию (БЕЗ ВОПРОСОВ)
   bash mkdir src/my-module

2. Создать main файл (БЕЗ ВОПРОСОВ)
   Write tool → my-module.js

3. Коммитить (БЕЗ ВОПРОСОВ)
   git add src/my-module/
   git commit -m "feat: создать новый модуль"

4. Пушить (БЕЗ ВОПРОСОВ)
   git push origin main

5. Сообщить результат:
   ✅ Модуль создан: src/my-module/
   ✅ Коммит: abc123def
   ✅ Запушено на GitHub
```

### Ситуация 2: Нужно удалить старый модуль

```
ПРАВИЛЬНО:
1. Переименовать модуль (БЕЗ ВОПРОСОВ)
   bash mv src/old-module src/old-module~

2. Коммитить (БЕЗ ВОПРОСОВ)
   git add src/
   git commit -m "feat: отключить старый модуль"

3. Пушить (БЕЗ ВОПРОСОВ)
   git push origin main

4. Сообщить результат:
   ✅ Модуль отключен: old-module~ (помечен тильдой)
   ✅ Коммит: def456abc
   ✅ Запушено на GitHub

НЕПРАВИЛЬНО:
❌ Удалить модуль полностью (rm -rf)
❌ Спросить: "Удалить модуль?"
❌ Оставить файл без пометки
```

### Ситуация 3: Нужно отредактировать файл

```
ПРАВИЛЬНО:
1. Отредактировать (БЕЗ ВОПРОСОВ)
   Read tool → посмотреть содержимое
   Edit tool → отредактировать

2. Коммитить (БЕЗ ВОПРОСОВ)
   git add src/module/file.js
   git commit -m "fix: исправить баг"

3. Пушить (БЕЗ ВОПРОСОВ)
   git push origin main

4. Сообщить результат:
   ✅ Файл отредактирован
   ✅ Коммит: ghi789jkl
   ✅ Запушено на GitHub
```
