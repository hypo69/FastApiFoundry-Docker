    Управление версиями

    Проект: FastAPI Foundry
    Версия документа: 0.6.0

    Схема версионирования

    Проект использует Semantic Versioning (MAJOR.MINOR.PATCH):

    Сегмент Когда увеличивается
    MAJOR Несовместимые изменения API или архитектуры
    MINOR Новые функции, обратно совместимые
    PATCH Исправления ошибок

    Версии оформляются как git-теги вида v0.6.0.

    Файл VERSION

    В корне проекта хранится файл VERSION с текущей версией:

    v0.6.0

    Этот файл:

    • читается scripts/Update-Project.ps1 для определения установленной версии;
    • обновляется автоматически при успешном обновлении;
    • используется как fallback, если git describe недоступен (offline-среда).

    Механизм обновлений

    Как работает

    При каждом запуске start.ps1 выполняется scripts/Update-Project.ps1:

    start.ps1
  └─ scripts/Update-Project.ps1
       ├─ Get-LocalVersion   → читает VERSION или git describe
       ├─ git fetch --tags   → получает теги с remote
       ├─ Compare-Versions   → сравнивает semver
       └─ Invoke-Update      → git checkout <tag> + install.ps1

    Условия запуска обновления

    Условие Поведение
    .git отсутствует Пропуск (не git-репозиторий)
    git не в PATH Пропуск с предупреждением
    Нет сети / remote недоступен Пропуск с предупреждением
    Версия актуальна Сообщение "✅ Версия актуальна"
    Доступна новая версия Интерактивный запрос [Y/n]

    Флаги Update-Project.ps1

    # Автоматическое обновление без запроса
.\scripts\Update-Project.ps1 -Silent

# Принудительное обновление (даже если версия совпадает)
.\scripts\Update-Project.ps1 -Force

    Выпуск новой версии (для разработчика)

    1. Обновить VERSION и заголовки файлов

    # Обновить файл VERSION
"v0.7.0" | Set-Content VERSION -Encoding UTF8

    Обновить версию в заголовках изменённых файлов:

    # Version: 0.7.0

    2. Обновить CHANGELOG.md

    ## [0.7.0] - 2025-12-15
### Added
- ...
### Fixed
- ...

    3. Создать git-тег и запушить

    git add .
git commit -m "release: v0.7.0"
git tag v0.7.0
git push origin main --tags

    Важно

    Тег должен быть создан после коммита с обновлённым VERSION.
    Иначе git describe вернёт предыдущий тег.

    Первый запуск (fresh install)

    При первом запуске start.ps1 файл venv/ отсутствует — скрипт автоматически вызывает install.ps1:

    start.ps1
  ├─ [Этап 0] Update-Project.ps1  ← проверка обновлений
  └─ [Этап 1] install.ps1         ← если venv не найден

    Порядок этапов гарантирует, что обновление проверяется до установки зависимостей — пользователь сразу получает актуальную версию.

    Ручное обновление

    Если автоматическое обновление не сработало:

    # Получить последний тег
git fetch --tags
git tag --sort=-version:refname | Select-Object -First 1

# Переключиться на нужный тег
git checkout v0.7.0

# Обновить зависимости
.\install.ps1

    Откат к предыдущей версии

    # Посмотреть доступные теги
git tag --sort=-version:refname

# Откатиться
git checkout v0.6.0
"v0.6.0" | Set-Content VERSION -Encoding UTF8
.\install.ps1

    Связанные файлы

    Файл Роль
    VERSION Текущая установленная версия
    CHANGELOG.md История изменений
    scripts/Update-Project.ps1 Скрипт проверки и применения обновлений
    start.ps1 Точка входа, вызывает Update-Project.ps1
    install.ps1 Установщик зависимостей, вызывается при обновлении
