Система тестирования Ai Assistant разделена на уровни для обеспечения надежности как отдельных функций, так и всей системы в целом.

## 👨‍💻 Руководство для разработчика: Стандарты безопасности

При добавлении новых функций или зависимостей разработчик обязан:
1.  **Проверять зависимости**: Перед коммитом новых пакетов в `requirements.txt` запустить `.\scripts\Invoke-Qa.ps1`.
2.  **Устранять уязвимости**: Использовать `.\scripts\Fix-Vulnerabilities.ps1` для автоматического патчинга.
    -   *Автоматизация:* Скрипт создаст ветку `security-fix`, применит исправления и автоматически создаст git-коммит.
3.  **Не допускать утечек**: Проверять отсутствие секретов в коде.
    -   *Защита:* В проект добавлен **pre-commit hook**. Скрипт `Invoke-Qa.ps1` автоматически устанавливает его в `.git/hooks/`.
    -   Хук блокирует попытку `git commit`, если в индексированных файлах найдены секреты.
3.  **Актуализировать тесты**: Любое изменение API должно сопровождаться обновлением соответствующих тестов в `tests/integration/`.

## 🧩 Уровни тестирования

### 1. Модульные тесты (Unit Tests)
Расположены в `tests/unit/`. Используют фреймворк **Pester** (PowerShell).
- **start.Tests.ps1**: Проверяет изолированную логику скрипта запуска:
  - Парсинг `.env` файлов и маскировку секретов.
  - Поиск портов Foundry и MkDocs.
  - Алгоритм выбора актуальной версии `llama.cpp`.

### 2. Интеграционные тесты (Integration Tests)
Расположены в `tests/integration/`. Используют **pytest** и **Pester**.
- **LlamaExtraction.Tests.ps1**: Проверяет физическую распаковку архивов и обновление конфигурации на диске.
- **test_fastapi_init.py**: Валидация серверной части:
  - **Конфигурация**: Корректность подстановки переменных окружения `${VAR}` в `config.json`.
  - **Ресурсы**: Проверка наличия минимум 5 ГБ свободного места для моделей.
  - **Права**: Тестирование возможности записи в рабочие директории (`~/.models`, `~/.rag`).

### 3. Покрытие API (API Coverage)
- **test_api.py**: Интеграционная проверка доступности и форматов ответов всех ключевых эндпоинтов сервера.
  - Использование **respx** для имитации ответов и обработки ошибок безопасности.
  - Валидация обработки некорректных запросов и таймаутов (404, 422, 504).

### 4. Нагрузочное тестирование (Performance)
- **locustfile.py**: Проверка способности сервера обрабатывать конкурентные запросы.
  - Сценарий: чтение списка моделей + генерация текста.
  - Запуск:
    ```powershell
    locust -f tests/integration/locustfile.py
    ```

## 🚀 Запуск проверок

Для запуска полного цикла QA (Linter + Types + Tests):
```powershell
.\scripts\Invoke-Qa.ps1
```

Результаты тестов и отчет о покрытии будут сохранены в `tests/reports/`.
