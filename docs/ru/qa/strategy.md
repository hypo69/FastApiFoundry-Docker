# Стратегия тестирования

**Версия:** 0.7.1 | **Проект:** AI Assistant (ai_assist)

---

## Уровни тестирования

### 1. Юнит-тесты (`tests/unit/`)

Изолированная проверка отдельных функций и классов без внешних зависимостей.

**Что покрывается:**

- Парсинг JSON (`src/utils/`) — корректные данные, пустые объекты, битый JSON, Unicode
- Обработка конфигурации (`config_manager.py`) — чтение секций, значения по умолчанию
- Утилиты (`src/utils/text_utils.py`, `src/utils/env_processor.py`)

**Правила:**

- Никаких сетевых вызовов
- Никаких обращений к файловой системе (кроме фикстур `tmp_path`)
- Все внешние зависимости заменяются моками через `pytest-mock`

---

### 2. Интеграционные тесты (`tests/integration/`)

Проверка взаимодействия между компонентами системы.

**Что покрывается:**

- PowerShell MCP сервер — выполнение команд через `subprocess`, передача JSON через stdin
- FastAPI эндпоинты — через `httpx.AsyncClient` с моком внешних сервисов (`respx`)
- Foundry AI — имитация ответов через `respx.mock`

**Правила:**

- Внешние HTTP-сервисы (Foundry, HuggingFace) всегда мокируются
- Реальный PowerShell допускается (Windows-окружение)
- Тесты должны проходить без запущенного сервера

---

### 3. Тесты агентов (`tests/agents/`)

Проверка логики AI-агентов с изоляцией от тяжёлых зависимостей.

**Что покрывается:**

- `RagAgent` — инъекция контекста, обработка Unicode, поведение при пустых результатах
- `PowerShellAgent` — формирование команд, обработка вывода
- Маршрутизация моделей по префиксу (`foundry::`, `hf::`, `llama::`, `ollama::`)

**Правила:**

- FAISS индекс всегда мокируется (`MagicMock`)
- LLM-клиенты всегда мокируются
- Проверяется логика агента, а не качество ответа модели

## Инструментарий

| Инструмент | Версия | Назначение |
|---|---|---|
| `pytest` | ≥ 8.0 | Запуск тестов |
| `pytest-asyncio` | ≥ 0.23 | Асинхронные тесты FastAPI |
| `pytest-cov` | ≥ 4.1 | Отчёт покрытия кода |

| `httpx` | ≥ 0.27 | HTTP-клиент для тестирования API |

Установка:

```powershell
venv\Scripts\pip.exe install -r requirements-qa.txt
```

## Конфигурация pytest

Файл `pytest.ini` в корне проекта:

```ini
[pytest]
testpaths = tests
norecursedirs = extensions .git .venv venv build dist
asyncio_mode = auto
addopts = -v --tb=short
```

- `testpaths = tests` — pytest ищет тесты только в `tests/`
- `norecursedirs` — исключает `extensions/` и служебные директории
- `asyncio_mode = auto` — все `async def` тесты запускаются автоматически

## Правила написания тестов

1. Файлы тестов — префикс `test_*.py`, классы — `Test*`, методы — `test_*`
2. Каждый класс и метод имеет docstring с описанием проверяемого сценария
3. Комментарии внутри тестов — только на английском
4. Guard clause вместо вложенных условий
5. Один тест — одна проверка (один `assert` на логическое утверждение)
6. Фикстуры для повторяющейся подготовки данных
