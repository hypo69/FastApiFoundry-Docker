# Смысл

## Что это такое

**AI Assistant** (`ai_assist`) — это оркестратор локальных AI моделей.

Одна программа, которая запускает AI модель прямо на вашем компьютере и даёт к ней
доступ через единый REST API. Никакого облака. Никаких серверов третьих сторон.
Ваши данные не покидают вашу машину — никогда.

При правильно настроенной политике администрирования ваши данные никогда не покинут
пределы учреждения. Правда, есть накладные расходы: железо должно быть достаточно
мощным, чтобы система была отзывчивой.

---

## Как это работает

AI Assistant не является AI моделью сам по себе. Это **оркестратор** — программа,
которая управляет несколькими AI движками и предоставляет к ним единый интерфейс.

```
Вы (браузер / скрипт / бот)
          │
          ▼
   AI Assistant
   REST API :9696
          │
   ┌──────┼──────────┬──────────┐
   ▼      ▼          ▼          ▼
Foundry  HuggingFace  llama.cpp  Ollama
(ONNX)  (PyTorch)   (GGUF/CPU)  (local)
```

Вы подключаете любое количество моделей и управляете ими из единого интерфейса.
Каждая модель доступна в любой момент — достаточно указать её в поле `model`
при запросе. Можно держать несколько моделей одновременно и направлять разные
задачи к разным моделям.

### Модели под разные задачи

Разные модели хорошо справляются с разными типами задач. Типичная конфигурация:

| Задача | Что нужно от модели | Пример модели |
|---|---|---|
| 💬 Ежедневный ассистент | Быстрый отклик, общий контекст | `foundry::qwen3-0.6b` |
| 🧠 Рассуждения и анализ | Глубокое понимание, длинный контекст | `ollama::deepseek-r1:7b` |
| 💻 Написание кода | Знание языков, точность синтаксиса | `ollama::codellama:7b` |
| 📄 Работа с документами | RAG + длинный контекст | `llama::D:/models/mistral-7b.gguf` |
| 🌍 Перевод и локализация | Многоязычность | `hf::facebook/nllb-200-distilled-600M` |
| 🔍 Суммаризация | Сжатие текста, выделение главного | `foundry::phi-3-mini` |

Все они работают через один и тот же API — вы просто меняете префикс в поле `model`.
Никакой переконфигурации сервера не нужно.

### Маршрутизация по префиксу

Модель выбирается по префиксу в поле `model`:

| Префикс | Движок | Пример |
|---|---|---|
| `foundry::` | Microsoft Foundry Local (ONNX) | `foundry::qwen3-0.6b` |
| `hf::` | HuggingFace Transformers | `hf::microsoft/phi-2` |
| `llama::` | llama.cpp (GGUF файл) | `llama::D:/models/qwen.gguf` |
| `ollama::` | Ollama | `ollama::llama3` |

## Универсальный доступ: любой клиент, любая сеть

Поскольку AI Assistant — это обычный HTTP-сервер с REST API, к нему можно обращаться
из любой программы, которая умеет делать HTTP-запросы:

| Клиент | Как подключается |
|---|---|
| Браузер | Встроенный веб-интерфейс на `http://localhost:9696` |
| Python / PowerShell скрипт | `requests.post("http://localhost:9696/api/v1/generate", ...)` |
| Telegram бот | Встроенный HelpDesk бот или свой через API |
| Браузерное расширение | Встроенное расширение-суммарайзер |
| Claude Desktop | Через MCP сервер |
| Любой другой MCP-клиент | Через MCP STDIO или HTTP протокол |

### Корпоративное развёртывание

AI Assistant можно развернуть на внутреннем сервере предприятия — и вся команда
получит доступ к AI через обычный HTTPS запрос внутри сети:

```
Сервер предприятия
  │  AI Assistant + модель
  │  https://ai.company.local:9696
  │
  ├─ Рабочее место 1  →  POST /api/v1/generate
  ├─ Рабочее место 2  →  POST /api/v1/generate
  ├─ Рабочее место 3  →  POST /api/v1/chat
  └─ Система документооборота  →  POST /api/v1/rag/search
```

Данные не покидают периметр сети. Модель работает на сервере, запросы идут по
внутренней сети — ничего не уходит наружу.

!!! warning "Не забывайте о безопасности"
    Если сервер доступен из сети — ограничьте доступ:

    - включите `API_KEY` в `.env` — все запросы будут требовать заголовок `X-API-Key`
    - настройте CORS в `config.json` — разрешите только доверенные домены
    - поставьте перед сервером reverse proxy (nginx) с TLS-сертификатом
    - не открывайте порт  9696 наружу без необходимости

---



!!! warning "Производительность зависит от железа"
    AI модели — вычислительно тяжёлые программы. Чем мощнее железо, тем быстрее ответ.

| Конфигурация | Что работает | Скорость ответа |
|---|---|---|
| 8 GB RAM, без GPU | Малые модели (0.5–1.5B параметров) | 5–15 сек |
| 16 GB RAM, без GPU | Средние модели (3–7B параметров) | 15–60 сек |
| 16 GB RAM + GPU 8 GB VRAM | Средние модели с GPU-ускорением | 2–10 сек |
| 32 GB RAM + GPU 16+ GB VRAM | Большие модели (13–34B параметров) | 3–15 сек |

!!! tip "Рекомендуемый минимум"
    - **RAM:** 16 GB
    - **CPU:** современный многоядерный (Intel 12th gen+ / AMD Ryzen 5000+)
    - **Диск:** SSD, от 20 GB свободного места под модели
    - **GPU:** опционально, но значительно ускоряет работу

!!! note "Квантованные модели (GGUF)"
    Формат GGUF позволяет запускать модели в сжатом виде с минимальными потерями качества.
    Модель 7B в формате Q4_K_M занимает ~4 GB RAM вместо ~14 GB в полном виде.
    Это основной способ запуска больших моделей на обычном железе.

## Безопасность данных: полная приватность

Это не просто удобство. Это принципиальное отличие от ChatGPT, Claude, Gemini
и любого другого облачного сервиса.

Когда вы отправляете документ в облачный AI — вы отправляете его на чужой сервер.
Юридически это означает передачу данных третьей стороне. Для многих профессий
это недопустимо по закону, по этике или по здравому смыслу.

**AI Assistant в отличие от ChatGPT, Claude, Gemini и любого другого облачного сервиса:**

- Вы выбираете модель в формате Microsoft Foundry, HuggingFace или llama.cpp (файлы GGUF)
- Все запросы обрабатываются локально
- Интернет не нужен совсем — ни для работы, ни для запросов
- Никакие данные никуда не отправляются

## Работа с архивами документов

Встроенная RAG-система позволяет загрузить любое количество документов,
проиндексировать их и задавать вопросы на естественном языке.

Поддерживается более 40 форматов: PDF, DOCX, XLSX, PPTX, HTML, изображения с текстом (OCR),
архивы ZIP и 7z, исходный код и многое другое.

Из этих данных собирается «облако смыслов» — векторный индекс, из которого модель извлекает точный ответ на ваш вопрос.

## Для кого это решение может быть полезным

### ⚖️ Адвокаты и юристы

Юридические документы — договоры, судебные решения, переписка, экспертизы —
содержат персональные данные клиентов и составляют адвокатскую тайну.
Загружать их в ChatGPT означает нарушать профессиональную этику и, во многих
юрисдикциях, закон.

С AI Assistant можно:

- Загрузить весь архив дела и задавать вопросы по нему
- Искать противоречия между показаниями в разных документах
- Быстро находить нужные пункты в многостраничных договорах
- Составлять черновики на основе своих шаблонов и прецедентов

Всё это — без единого байта, покинувшего ваш компьютер.

### 🏥 Врачи и медицинский персонал

Медицинские данные — история болезни, результаты анализов, снимки, выписки —
защищены законодательством во всех странах. Передача их в облако без явного
согласия пациента является нарушением.

С AI Assistant можно:

- Анализировать историю болезни пациента и задавать вопросы по ней
- Искать по архиву медицинской документации клиники
- Получать справки по протоколам лечения из загруженных руководств
- Работать с результатами исследований без риска утечки

### 🏢 Бухгалтеры и финансисты

Финансовая отчётность, налоговые декларации, банковские выписки — всё это
составляет коммерческую тайну. Ни один здравомыслящий финансовый директор
не должен загружать баланс компании в публичный AI-сервис.

С AI Assistant можно:

- Задавать вопросы по финансовым отчётам за несколько лет
- Искать расхождения в больших таблицах
- Анализировать договоры с контрагентами
- Работать с внутренними регламентами и инструкциями

### 🔬 Учёные и исследователи

Неопубликованные данные, черновики статей, результаты экспериментов —
это интеллектуальная собственность, которую нельзя передавать третьим сторонам
до публикации.

С AI Assistant можно:

- Работать с корпусом литературы по теме исследования
- Задавать вопросы по собственным данным и заметкам
- Анализировать большие массивы текстовых данных
- Использовать специализированные модели для конкретных областей

### 🏛️ Государственные структуры и силовые ведомства

Документы с грифом ограниченного доступа, внутренние регламенты, оперативные
материалы — по определению не могут обрабатываться на зарубежных серверах.

AI Assistant разворачивается в изолированной сети без выхода в интернет
и работает полностью автономно.

### 📚 Нотариусы, архивисты, историки

Работа с большими архивами документов — завещания, архивные дела, исторические
источники — требует инструмента, который умеет искать по тексту и отвечать
на вопросы. AI Assistant превращает папку с документами в базу знаний,
к которой можно обращаться на естественном языке.

### 💼 Корпоративные команды

Внутренняя документация компании — регламенты, инструкции, базы знаний,
переписка — не должна уходить за периметр. AI Assistant разворачивается
на внутреннем сервере, и вся команда получает доступ к корпоративному AI
без облачных рисков.

---

## Где имеет смысл использовать

- **Работа с конфиденциальными документами** — юридические, медицинские, финансовые архивы
- **Изолированные сети** — без интернета, в закрытом контуре
- **Корпоративная среда** — данные не покидают периметр
- **Локальная разработка** — быстрый старт без регистраций и оплаты
- **Исследования** — сравнение моделей, эксперименты с промптами
- **Автоматизация** — подключение к скриптам, ботам, браузерным расширениям

---

## Где не имеет смысла использовать

- Если нужна облачная масштабируемость на тысячи пользователей
- Если компьютер слабый — большие модели требуют RAM и желательно GPU
- Если нужны именно GPT-4, Claude или Gemini — облачные модели не входят в зону ответственности этой системы — здесь только локальные модели.

  Однако облачные модели не забыты. В составе проекта есть браузерное расширение
  (`extensions/browser-extension-summarizer`), которое умеет работать с Gemini, OpenAI-совместимыми
  API и OpenRouter. Это недокументированная возможность — она работает, но пока не
  входит в основной документацию.

## Как это выглядит на практике

Запускаете один раз:

```powershell
powershell -ExecutionPolicy Bypass -File .\start.ps1
```

Дальше у вас есть:

- **Веб-интерфейс** на `http://localhost:9696` — чат, управление моделями, RAG, настройки
- **REST API** на `http://localhost:9696/api/v1` — для своих приложений
- **Swagger UI** на `http://localhost:9696/docs` — интерактивная документация API
- **MCP серверы** — для подключения к Claude Desktop и другим MCP-клиентам
