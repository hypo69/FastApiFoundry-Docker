# Веб-интерфейс — справочник администратора

Веб-интерфейс доступен по адресу **http://localhost:9696** после запуска сервера.

Интерфейс состоит из 16 табов. Ниже описан каждый таб: что отображается, какие поля для чего, какие индикаторы состояния.

---

## Models — активные модели и статус системы

Открывается по умолчанию при входе.

### System Status

Блок в верхней части таба. Отображает текущее состояние всех сервисов: Foundry, HuggingFace, llama.cpp, Ollama, RAG. Обновляется при загрузке страницы.

### Active Models

Карточки загруженных в память моделей. Каждая карточка показывает:

- имя модели с префиксом провайдера (`foundry::`, `hf::`, `llama::`, `ollama::`)
- кнопку выгрузки модели из памяти

Кнопка **Refresh** — обновить список активных моделей.

### Available Models

Список всех доступных моделей, сгруппированных по провайдеру. Прокручиваемый список (макс. 520px). Кнопка **Refresh** справа в заголовке.

---

## Foundry — управление Microsoft Foundry Local

### Foundry Service

Заголовок карточки содержит два индикатора:

- **Port badge** (`foundry-port-indicator`) — показывает порт Foundry, например `Port: 63995`. Серый — порт неизвестен, синий — определён.
- **Status badge** (`foundry-service-status`) — текущий статус сервиса: `running` / `stopped` / `unknown`. Цвет: зелёный / красный / серый.

Элементы управления:

- **Переключатель** (toggle switch) — запустить / остановить Foundry сервис. Подпись меняется: `Start Foundry` / `Stop Foundry`.
- **Кнопка обновления** — проверить статус вручную.
- **foundry-service-info** — текстовая область под переключателем, показывает детали статуса.

!!! info "Внешний сервис"
    Foundry — внешний сервис Microsoft. Обычно запускается автоматически при старте `start.ps1` и не требует ручного управления.

### Downloaded Models

- **foundry-inactive-badge** — предупреждение (жёлтый alert), появляется если Foundry не запущен.
- Прокручиваемый список скачанных моделей (макс. 220px).
- Выпадающий список `foundry-downloaded-model-select` — выбор модели для загрузки.
- **Load & Use** — загрузить выбранную модель в память.
- **Refresh Downloaded** — обновить список скачанных моделей с диска.
- **foundry-load-status** — статус операции загрузки.

### Available Models

- **List Models for Download** — получить каталог моделей доступных для скачивания.
- **Include GPU models** (чекбокс) — включить GPU-модели в список.
- Прокручиваемый список `foundry-models-list` (макс. 320px).
- Информационный блок с инструкцией по CLI-скачиванию.

## HuggingFace — модели из HuggingFace Hub

### HuggingFace Hub

Два подтаба:

- **My Models** — модели из вашего аккаунта HuggingFace (требует токен).
- **Popular Public** — популярные публичные модели.

Кнопка **Load** — загрузить список с Hub. Список `hf-hub-list` (макс. 220px).

### Downloaded Locally

Список моделей, скачанных на диск. Кнопка **Refresh** — обновить список. `hf-downloaded-list` (макс. 260px).

### Loaded in Memory

Список моделей, загруженных в RAM. `hf-loaded-list` (макс. 100px).

### HuggingFace Status

Индикаторы состояния:

- наличие `transformers` и `torch`
- установлен ли токен (`hf_token_set`)
- директория моделей

Кнопка **Refresh** — обновить статус.

### Download Model

- **Model ID** — поле ввода идентификатора модели, например `google/gemma-2b`.
- **hf-token-warning** — предупреждение (жёлтый alert), появляется если токен не задан. Содержит ссылку на Settings → HuggingFace.
- **Download** — скачать модель. Токен берётся автоматически из Settings.
- **hf-download-status** — прогресс и статус скачивания.

## llama.cpp — GGUF модели на CPU/GPU

### GGUF Models in ~/.models

Список GGUF-файлов на диске. Кнопка **Refresh** — сканировать директорию.

### Server Status

- **llama-status-body** — состояние сервера: запущен / остановлен, PID, URL, активная модель.
- Кнопка обновления статуса.

### Service Control

- **Переключатель** — запустить / остановить llama.cpp сервер.
- **llama-start-status** — статус операции запуска.

### Parameters

Поля для настройки запуска сервера:

| Поле | Назначение |
|---|---|
| **Model Path (.gguf)** | Путь к файлу модели. Кнопки: открыть файловый браузер, сканировать диск. |
| **Port** | Порт сервера (по умолчанию 9780). |
| **Context** | Размер контекста в токенах (по умолчанию 4096). |
| **Threads** | Количество потоков CPU (0 = авто). |
| **GPU Layers** | Количество слоёв на GPU (0 = только CPU). |

- **Copy to ~/models** — скопировать выбранный файл в директорию моделей.
- **llama-openai-url** — URL OpenAI-совместимого API (`http://127.0.0.1:9780/v1`). Кнопка копирования.

!!! info "Ollama в этом же табе"
    Ниже в табе llama.cpp расположен раздел Ollama — список локальных моделей и Pull Model. Это дублирует отдельный таб Ollama.

## RAG — база знаний

### Active RAG Base

Текущий активный профиль RAG:

- имя профиля
- количество чанков
- модель эмбеддингов
- дата обновления

Кнопка **Refresh** — обновить статус.

### Index Documents

Два подтаба:

**ZIP / Files** — загрузить файлы для индексации:

- Поле выбора файлов (zip, 7z, rar, tar, pdf, docx, md и др.).
- **Upload & Index** — загрузить и проиндексировать.
- **rag-upload-status** — статус загрузки.

**Directory** — выбрать папку целиком:

- Поле выбора директории (`webkitdirectory`).
- **Index Directory** — проиндексировать все файлы в папке.
- **rag-dir-index-status** — статус индексации.

### RAG Bases

Список профилей RAG из `~/.rag/`. Каждый профиль отображается с кнопками активации и удаления. Кнопка **Refresh** — обновить список.

### Build New RAG Base

| Поле | Назначение |
|---|---|
| **Source Directories** | Список директорий-источников. Кнопка **Add Directory** открывает файловый браузер. |
| **Chunk size** | Размер текстового чанка в символах (по умолчанию 1000). |
| **Overlap** | Перекрытие между чанками (по умолчанию 50). |
| **Embedding model** | Модель для создания эмбеддингов: `all-mpnet-base-v2` (рекомендуется), `all-MiniLM-L6-v2` (быстрая), `paraphrase-multilingual`. |

Кнопки:

- **Build Index** — построить индекс.
- **Rebuild (force)** — принудительно пересобрать.
- **Создать индекс в Colab** — открыть Google Colab notebook для построения на GPU.

- **rag-build-status** — прогресс и результат построения.

!!! tip "Слабый компьютер"
    Для больших баз документов рекомендуется строить индекс в Google Colab (бесплатный GPU Tesla T4). Ускорение в 10–30× по сравнению с CPU.

### Test Search

- Поле ввода тестового запроса.
- **Search** — выполнить поиск по активному индексу.
- **rag-search-output** — результаты поиска (макс. 200px).

### Document Manager

Инкрементальное управление документами в RAG:

- **rag-doc-count** (badge) — количество документов в базе.
- **rag-doc-stats** — статистика: документов, активных чанков, векторов FAISS.
- Список документов `rag-documents-list` (макс. 320px).

Кнопки:

- **Refresh** — обновить список.
- **Compact** — удалить мёртвые векторы из FAISS (появляется при наличии неактивных чанков).
- **Add Document** — открыть модальное окно добавления документа.

Модальное окно **Add / Edit Document**:

| Поле | Назначение |
|---|---|
| **Title** | Заголовок документа. |
| **Content** | Текст документа. Счётчик символов `rag-doc-char-count`. |
| **Source path** | Путь к исходному файлу (опционально). |

### Text Extractor

Извлечение текста из файлов и URL для предпросмотра перед индексацией.

**File** подтаб:

- Поле выбора файла (40+ форматов: PDF, DOCX, XLSX, PPTX, изображения OCR, HTML, архивы, исходный код).
- **Extract** — извлечь текст.

**URL** подтаб:

- Поле ввода URL.
- **JavaScript** (чекбокс) — включить JS-рендеринг страницы.
- **OCR images** (чекбокс) — распознавать текст на изображениях.
- **Extract** — извлечь текст.

Результат:

- **ext-result-meta** — метаданные: имя файла, метод извлечения, количество символов.
- **ext-result-text** — извлечённый текст (readonly, 8 строк, растягивается).
- **Copy** — скопировать текст в буфер обмена.

## Chat — чат с AI моделью

### Область чата

- **chat-messages** — история сообщений. Пустое состояние: иконка + подсказка.
- **chat-input** — поле ввода сообщения.
- **Send** — отправить сообщение.
- **Stop** (кнопка с иконкой стоп) — прервать генерацию. Активна только во время генерации.
- **Clear** — очистить историю чата.

### Chat Settings (правая панель)

**Выбор модели:**

- `chat-model` — выпадающий список моделей. `Auto-select` — автовыбор.
- Подсказка: выбранная модель сохраняется как дефолтная.

**Индикаторы последнего ответа** (`chat-stats-badge`, появляется после первого ответа):

- **chat-model-badge** (синий badge) — имя использованной модели.
- **chat-timer** (серый badge) — время генерации, например `2.3s`.
- **chat-tokens-badge** (голубой badge) — количество токенов в ответе.

**Параметры генерации:**

| Поле | Назначение |
|---|---|
| **Temperature** | Слайдер 0–2, шаг 0.1. Значение отображается рядом (`temp-value`). |
| **Max Tokens** | Числовое поле, 1–8192. |
| **Use RAG Context** | Чекбокс — использовать RAG при генерации. |
| **Auto-translate dialog** | Чекбокс — автоматически переводить диалог. |
| **Your language** | Выпадающий список языков (16 языков). `Auto-detect` — определить автоматически. |

Информационный блок — инструкции по загрузке моделей через CLI и UI.

## API Keys — ключи доступа

### This Server API Key

Управление ключом доступа к этому FastAPI серверу.

- **local-api-key-status-badge** — статус: `Active` (зелёный) / `Not set` (серый).
- **local-api-key-preview** — поле с текущим ключом (скрыт по умолчанию).
  - Кнопка **Show/hide** — показать / скрыть ключ.
  - Кнопка **Copy** — скопировать в буфер.
- **Generate New Key** — сгенерировать новый ключ.
- **Remove Key** — удалить ключ (появляется если ключ установлен).
- **local-api-key-msg** — статус операции.

!!! info "Как использовать"
    После генерации добавьте ключ в каждый клиентский запрос как заголовок `X-API-Key: <key>` или параметр `?api_key=<key>`.

### Provider API Keys

Список ключей для внешних провайдеров (HuggingFace, OpenAI, DeepL и др.).

- `providers-list` — список провайдеров с полями ввода ключей.
- **Refresh** — обновить список.
- **Export** — скачать ключи в JSON для синхронизации с браузерным расширением.
- **Import** — загрузить JSON с ключами из браузерного расширения.

## Logs — просмотр логов

### Панель управления

| Элемент | Назначение |
|---|---|
| **log-file-select** | Выбор файла лога из списка доступных. |
| **log-level-filter** | Фильтр по уровню: All / DEBUG / INFO / WARNING / ERROR. |
| **log-search** | Поиск по тексту. Кнопка **×** — очистить поиск. |
| **log-lines-count** | Количество строк: 100 / 200 / 500 / 1000. |
| **Auto** (toggle) | Автообновление лога. |
| **Refresh** | Обновить вручную. |
| **Download** | Скачать файл лога. |
| **Clear** | Очистить файл лога. |

Настройки хранения (вторая строка панели):

| Поле | Назначение |
|---|---|
| **Lines per file** | Максимум строк в файле лога. |
| **Days to keep** | Срок хранения файлов лога в днях. |
| **Save** (кнопка) | Сохранить настройки хранения. |
| **log-dir-info** | Информация о директории логов. |

### Область вывода

- **log-output** — содержимое лога с подсветкой уровней.
- **log-status** — строка статуса: количество строк, фильтры.
- **Scroll to bottom** — прокрутить к последней строке.
- **Toggle word wrap** — включить / выключить перенос строк.
- **Load older** — загрузить более старые записи.
- **log-footer-info** — информация о загруженных строках.
