# Настройка подтверждения действий (Human-in-the-loop)

## Введение

При использовании AI-агентов для управления системой или базами данных возникают риски выполнения деструктивных команд (например, удаление файлов, изменение системных настроек или отправка писем). Механизм **Human-in-the-loop (HITL)** позволяет внедрить обязательное подтверждение пользователем критических операций перед их фактическим выполнением MCP-сервером.

## Как это работает

В архитектуре MCP подтверждение может быть реализовано на двух уровнях:
1.  **На стороне сервера:** Сервер возвращает запрос на подтверждение вместо выполнения.
2.  **На стороне оркестратора (рекомендуется):** Оркестратор анализирует метаданные инструментов и блокирует выполнение до получения сигнала от пользователя.

## Шаг 1: Маркировка опасных инструментов

При создании кастомного MCP-сервера (на Python или PowerShell) рекомендуется добавлять флаг в описание или схему инструмента. Хотя стандарт MCP активно развивается, хорошей практикой является использование префикса `danger_` или специальных метаданных в `description`.

### Пример на Python:
```python
@server.list_tools()
async def handle_list_tools() -> list[types.Tool]:
    return [
        types.Tool(
            name="delete_database_record",
            description="[DANGEROUS] Удаляет запись из базы данных. Требует подтверждения.",
            # ... schema ...
        )
    ]
```

## Шаг 2: Реализация перехвата в интерфейсе

Наш веб-интерфейс чата поддерживает перехват команд. Если в конфигурации агента или метаданных инструмента указано требование подтверждения:

1.  Агент формирует запрос на вызов инструмента (`call_tool`).
2.  Оркестратор видит маркер `[DANGEROUS]` или сверяется со списком запрещенных к авто-исполнению имен.
3.  В интерфейсе чата появляется карточка действия с кнопками **"Разрешить"** и **"Отклонить"**.
4.  Запрос отправляется на сервер только после нажатия кнопки пользователем.

## Шаг 3: Настройка списка ограничений

В `config.json` можно задать список инструментов, которые всегда требуют подтверждения, независимо от того, что передает сам сервер:

```json
{
  "mcp_agent": {
    "require_confirmation": [
      "delete_file",
      "terminate_process",
      "send_email",
      "execute_shell_command"
    ]
  }
}
```

## Рекомендации по безопасности

- **Никогда не доверяйте LLM** выполнение команд `rm`, `del`, `drop`, `format` без HITL.
- **Логируйте отказы:** Если пользователь отклонил действие, это должно фиксироваться в логах для последующего анализа поведения агента.
- **Таймауты:** Устанавливайте разумный таймаут на ожидание подтверждения. Если пользователь не ответил в течение 5 минут, действие должно быть автоматически отменено.

---
*Использование HITL — это баланс между автономностью и безопасностью. Начинайте с жестких ограничений, постепенно разрешая автоматизацию для проверенных инструментов.*
