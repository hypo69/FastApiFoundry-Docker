# 📚 RAG Система (Retrieval-Augmented Generation)

Модуль `src/rag/rag_system.py` является ядром системы RAG, отвечающим за управление векторными индексами FAISS, поиск релевантных фрагментов текста и их подготовку для передачи в LLM.

## 🧠 Основные компоненты

### `RAGSystem`

Класс `RAGSystem` управляет жизненным циклом RAG-индекса. Он реализован как синглтон, чтобы гарантировать единое состояние индекса во всем приложении.

#### `initialize()`
Асинхронный метод, вызываемый при старте приложения. Загружает индекс из директории, указанной в `config.rag_index_dir`. Если индекс не найден, RAG-функциональность будет отключена.

#### `reload_index(index_dir: str)`

**Назначение:** Динамическая перезагрузка индекса RAG. Позволяет переключать активные профили (базы знаний) без перезапуска всего FastAPI сервера.

**Логика:**
1.  Проверяет существование директории `index_dir` и файлов `faiss.index` и `chunks.json`.
2.  **Проверка целостности индекса (`_check_index_integrity`)**: Перед загрузкой `faiss.index` выполняется базовая проверка файла (существование, тип, минимальный размер). Это предотвращает попытки загрузки поврежденных или пустых файлов, которые могут привести к сбоям.
3.  **Проверка совместимости размерности векторов**: Сравнивает размерность (`.d`) загруженного FAISS индекса с размерностью эмбеддингов, генерируемых текущей `SentenceTransformer` моделью (`_get_model().get_sentence_embedding_dimension()`). Если размерности не совпадают, загрузка прерывается с ошибкой, предотвращая некорректную работу поиска.
4.  Загружает `faiss.index` (векторная база) и `chunks.json` (метаданные текстовых фрагментов).
5.  **Удаление дубликатов (`_remove_duplicate_chunks`)**: После загрузки чанков из `chunks.json` выполняется их фильтрация. Если несколько чанков содержат идентичный текст, остаются только уникальные. Это оптимизирует использование памяти и предотвращает выдачу повторяющихся результатов.
6.  Атомарно обновляет внутреннее состояние `self.index` и `self.chunks`.
7.  Очищает кеш поиска (`_search_cache`), чтобы при следующем запросе использовались данные из нового индекса.
8.  **Обновление `meta.json`**: Автоматически сохраняет или обновляет файл `meta.json` в директории индекса. В нем фиксируются такие параметры, как количество чанков, используемая модель и время последнего обновления.

#### `search(query: str, top_k: int)`

Выполняет векторный поиск по индексу.
1.  **Кеширование**: Проверяет, нет ли результата для данного запроса в кеше (`_search_cache`).
2.  **Генерация эмбеддингов**: Использует `SentenceTransformer` (`_get_model()`) для преобразования текстового запроса в вектор.
3.  **Поиск в FAISS**: Выполняет поиск ближайших соседей в `self.index`.
4.  **Форматирование результатов**: Связывает найденные индексы с оригинальными текстовыми фрагментами из `self.chunks` и возвращает их вместе с оценкой схожести.

#### `filter_by_source(results: List[Dict], sources: List[str])`

Фильтрует результаты поиска, оставляя только те, которые соответствуют заданному списку источников.

#### `filter_by_score(results: List[Dict], min_score: float)`

Отсекает результаты поиска, чья оценка схожести (`score`) ниже указанного порога.
