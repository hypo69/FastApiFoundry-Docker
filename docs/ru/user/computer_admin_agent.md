# ИИ-агент администратора компьютера

`computer_admin` — агент для диагностики и администрирования локального Windows-компьютера через FastAPI Foundry, PowerShell, Python и локальную модель.

Агент предназначен для сценария:

```text
запрос пользователя
  -> выбор агента
  -> сбор snapshot/time-series
  -> rule engine
  -> correlation + health score
  -> reasoning loop локальной модели
  -> понятный ответ
```

## Когда использовать

Используйте агента, когда нужно:

- проверить состояние Windows-компьютера;
- найти причины высокой нагрузки CPU/RAM;
- посмотреть диски, службы, процессы, автозагрузку;
- проверить Defender и Windows Update;
- собрать отчет `JSON`, `HTML`, `RST`;
- начать накопление истории состояния системы во времени;
- попросить модель составить дополнительный диагностический PowerShell или Python скрипт.

## Быстрый запуск через API

Если сервер уже запущен:

```text
http://localhost:9696
```

можно выполнить:

```powershell
$body = @{
  agent = "computer_admin"
  message = "Собери диагностику компьютера, оцени health score и объясни вероятные причины проблем"
  temperature = 0.2
  max_tokens = 2048
  max_iterations = 5
} | ConvertTo-Json

Invoke-RestMethod `
  -Uri "http://localhost:9696/api/v1/agent/run" `
  -Method POST `
  -Body $body `
  -ContentType "application/json"
```

## Автоматический выбор агента

Можно не указывать конкретного агента:

```json
{
  "agent": "auto",
  "message": "Проверь систему и объясни, что требует внимания"
}
```

Пока автономный диагностический агент один, режим `auto` выбирает `computer_admin`. В ответе API есть два поля:

| Поле | Значение |
|---|---|
| `agent` | что запросил клиент, например `auto` |
| `selected_agent` | фактически выбранный агент, например `computer_admin` |

## Основной диагностический инструмент

Главный инструмент агента — `invoke_os_diagnostic`.

Он запускает PowerShell toolkit:

```text
scripts/os_diagnostic/Invoke-OsDiagnostic.ps1
```

Toolkit собирает:

- ОС, uptime, CPU, RAM;
- диски и свободное место;
- топ процессов по CPU и памяти;
- службы;
- сетевые адаптеры;
- Microsoft Defender;
- Windows Update;
- события Windows Event Log;
- findings с severity;
- correlation signals;
- итоговый `health_score`.

Отчеты сохраняются в:

```text
logs/os_diagnostic/
```

Форматы:

| Файл | Назначение |
|---|---|
| `os_timeseries.jsonl` | история состояния во времени |
| `os_diagnostic_report.json` | машинный отчет для модели и API |
| `os_diagnostic_report.html` | отчет для просмотра в браузере |
| `os_diagnostic_report.rst` | строгий текстовый отчет |

## Скрипты, которые создает модель

Если стандартных инструментов недостаточно, модель может сама составить диагностический скрипт и выбрать язык:

| Язык | Когда выбирать |
|---|---|
| PowerShell | WMI/CIM, реестр, Event Log, службы, Windows-настройки |
| Python | анализ JSON/time-series, агрегации, scoring, отчеты, проверка файлов проекта |

Доступные инструменты:

| Инструмент | Назначение |
|---|---|
| `run_powershell_check` | выполнить временный PowerShell diagnostic-only скрипт |
| `run_python_check` | выполнить временный Python diagnostic-only скрипт |

Скрипты сохраняются во временную рабочую папку:

```text
logs/os_diagnostic/generated_scripts/
```

Есть базовая защита от очевидных изменяющих операций: удаление файлов, остановка служб, завершение процессов, `eval/exec`, сетевые загрузки и другие опасные паттерны блокируются.

## Примеры запросов

```text
Проверь состояние компьютера за последние 24 часа и объясни health score.
```

```text
Собери диагностику без Event Log, затем Python-скриптом найди топ процессов по памяти из отчета.
```

```text
Проверь ошибки Windows Update и предложи план verify/fix без применения изменений.
```

```text
Составь PowerShell диагностический скрипт для проверки служб Hyper-V и объясни результат.
```

## Безопасность

По умолчанию агент ориентирован на диагностику. Действия, которые меняют систему, должны выполняться только по явной просьбе пользователя.

Для очистки временных файлов сначала используется `dry_run=true`, чтобы показать, что будет удалено.

Для production-сценариев рекомендуется запускать сервис под отдельным пользователем с минимальными правами.
