# Веб-интерфейс FastAPI Foundry

После запуска `.\start.ps1` откройте в браузере **http://localhost:9696**

Интерфейс — одностраничное приложение (SPA) с вкладками.
Индикатор в шапке показывает статус подключения:

| Индикатор | Значение |
|---|---|
| 🟢 Connected | API и Foundry работают |
| 🟡 API Only | API работает, Foundry недоступен |
| 🔴 Error / Offline | сервер недоступен |

При загрузке страница автоматически запрашивает `/api/v1/config`, `/api/v1/health`
и список моделей. Статус обновляется каждые 30 секунд.

---

## 💬 Chat

Основная вкладка для общения с моделью.

- Поле ввода, отправка по `Enter` (новая строка — `Shift+Enter`)
- Выбор модели из списка (Foundry, `hf::...`, `llama::...`)
- Регулировка **Temperature** (0–2) и **Max Tokens**
- Чекбокс **Use RAG Context** — добавляет релевантный контекст из документации
- Выбранная модель автоматически сохраняется как `default_model` в `config.json`
- Выбор **Locale** — язык ответа (русский, иврит, французский и др.)

---

### Workflow диалога

Каждое сообщение проходит следующий путь:

```
Пользователь пишет сообщение (на любом языке)
        │
        ▼  [1] Перевод входящего сообщения → English
        │   (translator.translate_for_model, если translator.enabled=true)
        │   Если source_lang="en" — пропускается
        │
        ▼  [2] RAG поиск (если use_rag=true)
        │   Запрос к векторному индексу FAISS
        │   Релевантные фрагменты добавляются в начало промпта
        │
        ▼  [3] Сборка промпта
        │   System prompt + RAG context + история сессии + сообщение
        │
        ▼  [4] Генерация ответа моделью
        │   foundry_client.generate_text() / generate_stream()
        │   Модель отвечает на English
        │
        ▼  [5] Обратный перевод ответа
        │   translator.translate_response(locale или source_lang)
        │   Если locale/source_lang="en" — пропускается
        │
        ▼  [6] Ответ пользователю (на его языке)
```

**Параметры перевода:**

| Параметр | Значение | Описание |
|---|---|---|
| `source_lang` | `"auto"` | Язык входящего сообщения. `"auto"` = автоопределение |
| `locale` | `""` | Язык ответа: `"ru"`, `"he"`, `"fr"` и т.д. Переопределяет `source_lang` |

Перевод включается через `config.json → translator.enabled = true`.

## 🤖 Models

Список моделей, доступных через все подключённые бэкенды.

- Карточка каждой модели: ID, провайдер, статус, максимум токенов
- **Use in Chat** — переключиться на эту модель во вкладке Chat
- **Test** — отправить тестовый запрос `"Hello!"` и показать ответ
- **Remove** — выгрузить модель из памяти
- **Use as default model** — сохранить в `config.json`
- **Add Model** — форма добавления (Foundry, HuggingFace, llama.cpp, Ollama, Custom)

---

## ⬇️ Foundry

Управление сервисом Microsoft Foundry Local.

- Кнопки **Start** / **Stop** Foundry
- Индикатор статуса и порта (определяется автоматически)
- Лог вывода Foundry в реальном времени
- Список загруженных моделей с кнопками использовать / удалить
- Выпадающий список для загрузки новой модели + прогресс-бар
- Прогресс проверяется каждые 5 секунд, таймаут — 5 минут

---

## 🤗 HuggingFace

Управление моделями с HuggingFace Hub.

- Список скачанных моделей с размерами и статусом (загружена / не загружена)
- Список популярных публичных моделей для быстрого скачивания
- **Download** — скачать модель через `snapshot_download`
- **Load** — загрузить в память для inference
- **Unload** — выгрузить из памяти (освободить RAM/VRAM)
- Статус интеграции: версии `transformers`, `torch`, наличие CUDA

---

## 🦙 llama.cpp

Управление локальным сервером llama.cpp.

- Статус сервера (запущен / остановлен)
- **Start** — запустить с выбранной GGUF моделью
- **Stop** — остановить сервер
- Список доступных GGUF файлов из `GGUF_SEARCH_DIRS`

---

## 🔍 RAG

Управление системой поиска по документации.

- Статус: включена / выключена, количество проиндексированных фрагментов
- Настройки: директория индекса, размер чанка, модель эмбеддингов, Top K
- **Rebuild Index** — переиндексировать документы
- **Clear All Chunks** — удалить весь индекс (с подтверждением)
- **Test Search** — проверить поиск, показывает результаты с оценкой релевантности

## ⚙️ Settings

Редактирование `config.json` через веб-форму.

| Секция | Параметры |
|---|---|
| FastAPI Server | хост, порт, режим, воркеры, автопоиск порта |
| Foundry AI | URL бэкенда, модель по умолчанию, temperature, max tokens, автозагрузка |
| RAG System | включить/выключить, директория индекса, размер чанка |
| Security | API ключ, HTTPS |
| Logging | уровень (DEBUG/INFO/WARNING/ERROR), путь к файлу |

Кнопки **Reload** (загрузить из файла) и **Save All** (сохранить изменения).

## 📋 Logs

Просмотр логов сервера в реальном времени.

- Фильтр по уровню: All / Debug / Info / Warning / Error
- Боковая панель: статус системы, сводка ошибок и предупреждений
- Кнопки обновить и очистить вид

## 🤖 Agent

Интерфейс для работы с AI агентами.

- Выбор агента из списка зарегистрированных
- Просмотр доступных инструментов агента
- Отправка задачи агенту с выбором модели
- Модель должна поддерживать function calling (Qwen, DeepSeek, Mistral)
