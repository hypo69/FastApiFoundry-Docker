# Веб-интерфейс FastAPI Foundry

FastAPI Foundry предоставляет интуитивно понятный веб-интерфейс, который позволяет легко взаимодействовать с локальными моделями ИИ, управлять ими и использовать функции, такие как чат и система RAG (Retrieval-Augmented Generation).

После запуска сервера FastAPI Foundry, вы можете получить доступ к веб-интерфейсу, открыв в браузере [http://localhost:9696](http://localhost:9696) (порт может отличаться в зависимости от настроек).

## 🗺️ Обзор интерфейса

Веб-интерфейс обычно состоит из нескольких ключевых разделов или вкладок, предоставляющих различную функциональность:

*   **Главная страница / Чат**: Основной экран для взаимодействия с ИИ-моделями.
*   **Выбор модели**: Раздел для просмотра доступных моделей и выбора активной.
*   **Настройки / Конфигурация**: Страница для изменения параметров приложения и моделей.
*   **RAG (Retrieval-Augmented Generation)**: Управление системой поиска и извлечения контекста.
*   **Агенты**: Интерфейс для взаимодействия с ИИ-агентами.

## 💬 Чат и Генерация текста

Основной функцией веб-интерфейса является возможность вести диалог с выбранной моделью ИИ или получать одноразовые ответы на запросы. В этом разделе вы сможете:

*   **Вводить свои запросы (промпты)**: Отправляйте текст модели для получения ответов.
*   **Выбирать модель**: Переключайтесь между доступными моделями Foundry Local, llama.cpp или Ollama.
*   **Настраивать параметры генерации**: Изменять температуру (креативность), максимальное количество токенов и другие параметры, влияющие на ответ ИИ.
*   **Использовать контекст RAG**: Если система RAG активна, релевантный контекст из ваших документов будет автоматически использоваться для формирования более точных и полных ответов.

## ⚙️ Управление моделями

В соответствующем разделе (или выпадающем списке на странице чата) вы можете:

*   **Просмотреть список доступных моделей**: Отображает все модели, которые обнаружил FastAPI Foundry.
*   **Загружать/Выгружать модели (для Foundry)**: Для бэкенда Foundry Local можно динамически управлять загрузкой моделей в память.

## 📚 Система RAG (Retrieval-Augmented Generation)

Раздел RAG позволяет управлять системой поиска и извлечения контекста:

*   **Просмотр статуса RAG**: Узнайте, активна ли система RAG, какая модель эмбеддингов используется, и сколько документов проиндексировано.
*   **Перестроение индекса**: Запустите процесс переиндексации ваших документов, если они были обновлены или добавлены новые.
*   **Поиск по индексу**: Проверьте, какие документы или фрагменты текста система RAG находит по вашему запросу.

## 🤖 Взаимодействие с агентами

Если в системе настроены ИИ-агенты, в веб-интерфейсе может быть доступен специальный раздел для их использования. Здесь вы сможете:

*   **Выбирать агента**: Выберите агента, который будет выполнять задачу.
*   **Отправлять команды**: Агент будет использовать свои инструменты для выполнения запроса.
*   **Просматривать ход выполнения**: Некоторые интерфейсы могут показывать последовательность действий агента и вызовы инструментов.

## 🛠️ Настройки приложения

Через веб-интерфейс или через файл `config.json` можно настроить различные параметры приложения, включая:

*   Порт сервера FastAPI.
*   Настройки логирования.
*   Параметры RAG системы.
*   Параметры подключения к Foundry Local.

Более детальная информация о конфигурации доступна в файле `config.json` и документации разработчика.
