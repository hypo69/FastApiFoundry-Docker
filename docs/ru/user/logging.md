# Логирование — руководство пользователя

## Где смотреть логи

### Веб-интерфейс (рекомендуется)

Откройте браузер: **http://localhost:9696** → вкладка **Logs**

Там доступно:

- выбор файла лога
- фильтрация по уровню (DEBUG / INFO / WARNING / ERROR)
- текстовый поиск с подсветкой
- автообновление каждые 5 секунд
- постраничная загрузка старых записей
- скачивание и очистка файла

### Файлы на диске

Все логи хранятся в папке `logs/` в корне проекта:

| Файл | Что содержит |
|---|---|
| `logs/fastapi-foundry.log` | Все события: запуск, запросы, модели, RAG |
| `logs/fastapi-foundry-errors.log` | Только ошибки (ERROR и выше) |
| `logs/fastapi-foundry-structured.jsonl` | Структурированные JSON-события |
| `logs/app.log` | Лог запуска сервера (uvicorn) |

Ротация файлов происходит автоматически:

| Файл | Макс. размер | Резервных копий |
|---|---|---|
| `fastapi-foundry.log` | 10 МБ | 5 |
| `fastapi-foundry-errors.log` | 5 МБ | 3 |
| `fastapi-foundry-structured.jsonl` | 20 МБ | 3 |

## Как пользоваться просмотрщиком логов

### Панель инструментов

| Элемент | Назначение |
|---|---|
| Выпадающий список файлов | Переключение между лог-файлами |
| All levels / DEBUG / INFO / WARNING / ERROR | Фильтр по уровню важности |
| Поле поиска | Текстовый поиск, подсвечивает совпадения |
| 100 / 200 / 500 / 1000 lines | Количество строк на странице |
| Auto (переключатель) | Автообновление каждые 5 секунд |
| ↻ (кнопка) | Обновить вручную |
| ↓ (кнопка) | Скачать текущий файл |
| 🗑 (кнопка) | Очистить текущий файл |

### Навигация в окне лога

- Кнопка **↓** в заголовке окна — прокрутить к последней строке
- Кнопка **⏎** — включить/выключить перенос длинных строк
- Кнопка **Load older** внизу — загрузить более старые записи

## Как читать логи в терминале

```powershell
# Следить за основным логом в реальном времени
Get-Content logs\fastapi-foundry.log -Wait -Tail 50

# Следить только за ошибками
Get-Content logs\fastapi-foundry-errors.log -Wait -Tail 20
```

## Формат строки лога

```
2026-04-19 18:30:00 | INFO     | fastapi-foundry     | lifespan            | 35   | ✅ RAG system initialized
```

| Поле | Значение |
|---|---|
| Дата/время | Когда произошло событие |
| Уровень | DEBUG / INFO / WARNING / ERROR / CRITICAL |
| Имя логгера | Модуль, который записал событие |
| Функция | Имя функции |
| Строка | Номер строки в коде |
| Сообщение | Текст события |

---

## Уровни важности

| Уровень | Что означает | Нужно ли действовать |
|---|---|---|
| `DEBUG` | Детальная отладка | Нет |
| `INFO` | Нормальная работа | Нет |
| `WARNING` | Нестандартная ситуация, но работает | Обратить внимание |
| `ERROR` | Ошибка, функция не выполнена | Да |
| `CRITICAL` | Критический сбой | Немедленно |

---

## Изменение уровня логирования

В файле `.env`:

```env
LOG_LEVEL=DEBUG   # DEBUG | INFO | WARNING | ERROR
```

Перезапустить сервер после изменения.

---

## Типичные сообщения и их значение

| Сообщение | Значение |
|---|---|
| `✅ RAG system initialized` | RAG готов к работе |
| `⚠️ RAG system not initialized` | RAG не загружен — поиск по документам недоступен |
| `✅ Auto-loading default model: ...` | Модель загружается автоматически при старте |
| `⚠️ Foundry is unavailable` | Foundry не запущен — AI-функции отключены |
| `❌ Connection failed` | Ошибка подключения к модели или сервису |
