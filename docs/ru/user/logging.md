# Логирование — руководство пользователя

## Что попадает в логи

Система логирования теперь ориентирована на диагностику проблем. В файлы пишутся
только предупреждения и ошибки:

- `WARNING`
- `ERROR`
- `CRITICAL`

Успешные HTTP-запросы, включая обычные ответы `200 OK`, в файловые логи не
попадают.

## Где смотреть логи

### Веб-интерфейс

Откройте:

```text
http://localhost:9696
```

Перейдите на вкладку **Logs**.

Там доступны:

- выбор файла лога
- фильтр по уровню
- поиск по тексту
- автообновление
- загрузка старых строк
- скачивание файла
- очистка выбранного файла
- настройка количества строк на файл
- настройка количества суток хранения

### Файлы на диске

Файлы хранятся в пользовательской temp-папке:

```powershell
$env:TEMP\aissistant
```

На Windows это обычно:

```text
C:\Users\<user>\AppData\Local\Temp\aissistant
```

Формат имени файла:

```text
aiassistant-YYYY-MM-DD-NNN.log
```

Пример:

```text
aiassistant-2026-05-07-001.log
```

## Ротация и срок жизни

Лог экспортируется в отдельные дневные файлы. Внутри одного дня файл также
делится по лимиту строк:

- `aiassistant-2026-05-07-001.log`
- `aiassistant-2026-05-07-002.log`
- `aiassistant-2026-05-07-003.log`

Если файл достиг лимита строк, система открывает следующий файл за тот же день.
Старые файлы удаляются по настройке количества суток хранения.

## Настройки во вкладке Logs

| Настройка | Назначение |
|---|---|
| `Lines per file` | Максимальное количество строк в одном файле |
| `Days to keep` | Сколько суток хранить дневные файлы |
| Save | Сохранить настройки в `config.json` |

После сохранения настройки применяются к текущему процессу.

## Настройки в config.json

```json
{
  "logging": {
    "level": "WARNING",
    "log_dir": "",
    "max_lines_per_file": 5000,
    "retention_days": 7
  }
}
```

Если `log_dir` пустой, используется `%TEMP%/aissistant`.

## Быстрый просмотр в PowerShell

```powershell
# Последние файлы
Get-ChildItem "$env:TEMP\aissistant" | Sort-Object LastWriteTime -Descending

# Смотреть свежий лог в реальном времени
$latest = Get-ChildItem "$env:TEMP\aissistant" -Filter "aiassistant-*.log" |
  Sort-Object LastWriteTime -Descending |
  Select-Object -First 1
Get-Content $latest.FullName -Tail 50 -Wait
```

## Формат строки

```text
2026-05-07 21:42:55 | WARNING  | src.api.app | lifespan | 56 | RAG system not initialized
```

| Поле | Значение |
|---|---|
| Дата и время | Когда произошло событие |
| Уровень | `WARNING`, `ERROR`, `CRITICAL` |
| Логгер | Модуль, который создал запись |
| Функция | Функция, где появилась запись |
| Строка | Номер строки в исходном файле |
| Сообщение | Описание события |

## Типичные записи

| Запись | Что означает |
|---|---|
| `HTTP POST /api/... -> 404` | Клиент вызвал несуществующий или недоступный endpoint |
| `Unhandled request exception` | Исключение дошло до FastAPI middleware |
| `Model generation failed` | Backend модели вернул ошибку |
| `Chat completion crashed` | Диалог в чате породил исключение |
| `RAG enhancement failed` | RAG-контекст не был добавлен, но запрос может продолжить работу |
