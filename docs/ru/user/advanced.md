# Продвинутые возможности

Этот раздел для пользователей, которые хотят тонко настроить работу системы.

---

## Параметры модели

### Temperature

Управляет случайностью ответа:

| Значение | Поведение |
|---|---|
| `0.0–0.3` | Детерминированный, точный, повторяемый ответ |
| `0.4–0.7` | Баланс точности и разнообразия (рекомендуется) |
| `0.8–1.2` | Творческий, разнообразный ответ |
| `1.3–2.0` | Непредсказуемый, экспериментальный |

### Max Tokens

Максимальная длина ответа в токенах (≈ 0.75 слова на токен):

| Значение | Примерная длина ответа |
|---|---|
| 256 | Короткий ответ, 1–2 абзаца |
| 512 | Средний ответ |
| 2048 | Развёрнутый ответ |
| 4096+ | Длинный документ, код |

## Несколько RAG индексов

Если администратор создал несколько баз знаний (профилей RAG), вы можете переключаться между ними:

1. Вкладка **RAG** → список профилей
2. Выберите нужный профиль → **Activate**
3. Активный профиль используется во всех запросах с включённым RAG

---

## Агенты

Вкладка **Agent** — запуск AI агентов, которые умеют выполнять многошаговые задачи с использованием инструментов (поиск, выполнение кода, работа с файлами).

1. Выберите агента из списка
2. Посмотрите доступные инструменты агента
3. Выберите модель (должна поддерживать function calling: Qwen, DeepSeek, Mistral)
4. Введите задачу и запустите

!!! warning "Требования к модели"
    Агенты работают только с моделями, поддерживающими function calling.
    Если агент не отвечает — попробуйте другую модель.

## Язык ответа

Параметр **Locale** в чате задаёт язык ответа независимо от языка вопроса:

- Пусто — модель отвечает на языке вопроса
- `ru` — всегда по-русски
- `en` — всегда по-английски
- `he` — иврит

Перевод работает только если администратор включил `translator.enabled` в конфигурации.
