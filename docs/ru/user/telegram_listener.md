# Telegram Слушатель

Telegram Listener Bot — пассивный бот, который **только слушает** входящие сообщения и сохраняет их в лог. Он никогда не отвечает пользователям.

## Для чего нужен

- Сбор реальных вопросов пользователей для последующей индексации в RAG
- Мониторинг активности в Telegram-каналах и группах
- Формирование датасетов для дообучения моделей
- Аудит входящего трафика без вмешательства в диалог

## Настройка

### 1. Создать бота через @BotFather

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot`
3. Задайте имя и username (должен заканчиваться на `bot`)
4. Скопируйте полученный токен

### 2. Добавить токен в `.env`

```env
TELEGRAM_LISTENER_TOKEN=123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 3. Добавить бота в нужный чат или канал

Если нужно слушать группу или канал — добавьте бота как участника.  
Для личных сообщений достаточно написать боту напрямую.

## Что сохраняется

Каждое входящее сообщение записывается в файл:

```
~/.aiassistant/dialogs/listener_log.jsonl
```

Формат одной записи:

```json
{
  "ts": "2025-01-15T14:32:10.123456",
  "chat_id": 123456789,
  "chat_type": "private",
  "user_id": 987654321,
  "username": "ivan_petrov",
  "content_type": "text",
  "text": "Как подключить RAG к своей модели?"
}
```

Поддерживаемые типы контента: `text`, `photo`, `document`, `voice`, `video`, `sticker`, `location`, `contact`, `audio`.

## Использование лога для RAG

Накопленные вопросы можно проиндексировать в RAG:

```bash
POST /api/v1/rag/index
# загрузить listener_log.jsonl как документ
```

Или через веб-интерфейс: таб **RAG** → **Загрузить файл** → выбрать `listener_log.jsonl`.

## Запуск

Бот запускается автоматически вместе с FastAPI сервером при наличии токена `TELEGRAM_LISTENER_TOKEN`.

Если токен не задан — бот молча не запускается, ошибок нет.
