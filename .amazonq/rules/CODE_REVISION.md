# 🔍 CODE REVISION RULES

**Version:** 0.7.1
**Project:** AI Assistant (ai_assist)

---

## Когда делать ревизию

Периодически (при каждом значительном изменении кода) и по запросу пользователя.

---

## Чеклист ревизии

### 1. Мёртвый код
- [ ] Файлы не подключённые в `app.py` → пометить `~`
- [ ] Функции/классы нигде не вызываемые → удалить или пометить
- [ ] Закомментированный код без объяснения → удалить
- [ ] Дублирующие классы/функции в одном файле → удалить старый

### 2. Аннотации типов
- [ ] Все функции имеют `-> ReturnType`
- [ ] Все параметры аннотированы
- [ ] `**kwargs` имеет тип `**kwargs: object` или конкретный тип
- [ ] Async generators: `-> AsyncIterator[T]`
- [ ] `None`-возвращающие функции: `-> None`

### 3. Docstrings
- [ ] Каждая публичная функция/метод/класс имеет docstring
- [ ] Формат: Google style (Args / Returns / Raises / Example)
- [ ] `Raises: Same as ...` → заменить на явные типы исключений
- [ ] Примеры: минимум 1, все варианты вызова если их несколько
- [ ] Workflow-цепочки: описать в виде ASCII-диаграммы в docstring или заголовке

### 4. Комментарии
- [ ] Комментарии внутри кода — только на английском
- [ ] Объясняют *почему*, не *что*
- [ ] Нет устаревших комментариев (ссылки на удалённый код)

### 5. Документация
- [ ] При изменении endpoint → обновить `docs/ru/dev/api_reference.md`
- [ ] При изменении конфига → обновить `docs/ru/dev/architecture.md`
- [ ] Версия в заголовке файла → 0.7.1
- [ ] CHANGELOG.md → добавить запись

### 6. Версионирование
- [ ] Заголовок файла: `Version: 0.7.1`
- [ ] `Changes in 0.7.1:` — описать изменения
- [ ] `VERSION` файл в корне актуален

## Команды для ревизии

```powershell
# Найти файлы без аннотаций возврата
findstr /rn "async def\|    def " src\**\*.py | findstr /v "->"

# Найти файлы с дублирующими классами
findstr /n "^class " src\**\*.py

# Проверить мёртвые файлы (не импортированные в app.py)
findstr /n "from .endpoints" src\api\app.py
```

1. `src/models/` — AI клиенты (foundry, hf, ollama, router)
2. `src/api/endpoints/` — все endpoint файлы
3. `src/rag/` — RAG система
4. `src/utils/` — утилиты
