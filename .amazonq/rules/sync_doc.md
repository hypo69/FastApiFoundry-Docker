## 🔴 ОБЯЗАТЕЛЬНО ПОСЛЕ КАЖДОГО ИЗМЕНЕНИЯ КОДА

После изменения **любого** файла с кодом автоматически:

### 1. Docstring / Docblock
- Обновить docstring изменённой функции/класса
- Если добавлен новый параметр — добавить в `Args:` / `@param`
- Если изменился возврат — обновить `Returns:` / `@returns`

### 2. CHANGELOG.md
- Добавить запись в формате:
  ```
  ## [0.6.0] - YYYY-MM-DD
  ### Changed
  - <описание изменения>
  ```

### 3. API Reference (если изменился endpoint)
- Обновить `docs/ru/dev/api_reference.md`
- Указать новые/изменённые параметры запроса и ответа

### 4. README.md (если изменилась архитектура или конфигурация)
- Обновить соответствующий раздел

## 📌 Порядок слов в заголовках

В заголовках, названиях разделов и подписях всегда использовать порядок **прилагательное + существительное**:

| Неправильно | Правильно |
|---|---|
| Порт динамический | Динамический порт |
| Ключ секретный | Секретный ключ |
| Индекс векторный | Векторный индекс |
| Сервер локальный | Локальный сервер |

---



При составлении любой документации **обязательно** использовать шаблоны:

### ⚠️ Предупреждение / информация

```markdown
!!! warning "Заголовок"
    Текст предупреждения или важной информации.
```

Использовать для: неочевидных моментов, ограничений, особенностей поведения.

### ❗ Опасность

```markdown
!!! danger "Заголовок"
    Текст о серьёзной опасности.
```

Использовать для: потери данных, утечки секретов, необратимых действий, угроз безопасности.

### Правило выбора

| Ситуация | Шаблон |
|---|---|
| Особенность, ограничение, неочевидное поведение | `!!! warning` |
| Потеря данных, утечка секретов, необратимые действия | `!!! danger` |
| Полезный совет, рекомендация | `!!! tip` |
| Нейтральная дополнительная информация | `!!! info` |
| Успешное завершение, подтверждение | `!!! success` |

## 🔴 CHANGELOG — ВСЕГДА АКТУАЛЕН

**Один файл `CHANGELOG.md` в корне проекта. Обновляется АВТОМАТИЧЕСКИ после каждого изменения кода — БЕЗ ИСКЛЮЧЕНИЙ.**

- Каждое изменение файла → запись в `/CHANGELOG.md` (корень проекта)
- Указывать компонент в описании:
  ```
  ## [0.6.0] - YYYY-MM-DD
  ### Added / Changed / Fixed / Removed
  - `src/utils/translator.py` — <описание>
  - `static/js/i18n.js` — <описание>
  ```
- Не накапливать изменения — писать сразу
- Не коммитить без записи в CHANGELOG
