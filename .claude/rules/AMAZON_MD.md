# 🧩 3.3. Взаимосвязи между компонентами, которые Модель обязана учитывать

## 🔵 WordPress ⇄ MCP Servers

Модель должна понимать:

* MCP WordPress Server (WP-CLI) работает поверх SSH агента
* все операции через CLI должны учитывать multisite и особенност нашего проекта (вместо нумерации в базе данных таблиц мультисайта используется двухбуквенно название языка. Например, wp_en_<има таблицы>,wp_de_<има таблицы>, ...)
* REST API и MCP взаимодействуют
* некоторые плагины используют RPC через CLAUDE

## 🔵 WordPress ⇄ PHP плагины

Модель должна знать:

* хуки, фильтры, приоритеты
* структуру MU-плагинов
* особенности Multisite
* что плагины `Ai Assistant-*`, А также `show-all-media`,`multisite-server-logs`,`CLAUDE-spam-filter`, `CLAUDE-ai` должны быть сетевыми

## 🔵 WordPress ⇄ база данных MariaDB

Модель обязана понимать:

* таблицы WordPress
* таблицы Multisite
* названия таблиц содержат **языковой префикс, а не ID**
* структура таблиц пользователей нестандартная

Пример:

```
wp_ru_users
wp_en_users
wp_he_users
```

## 🔵 MCP ⇄ PowerShell

Модель должна понимать:

* PowerShell используется для локальной автоматизации
* запускается как STDIO MCP Server

## 🔵 REST API и Сетевая архитектура

Модель обязана помнить:
* Основной и единственный интерфейс взаимодействия: **REST API**
* Base URL: `http://localhost:9696/api/v1`
* Основные эндпоинты:
    * `POST /api/v1/generate` — генерация контента.
    * `POST /api/v1/chat` — диалоговый интерфейс.
    * `POST /api/v1/rag/search` — поиск по базе знаний (RAG).

## 🔵 MCP ⇄ Python

Модель должна учитывать:

* Python версии 3.12+
* серверы выполняют логику WordPress, MariaDB, Huggingface, CLAUDE
* конфиги лежат в JSON
