    Извлечение текста (Text Extractor)

    Text Extractor — встроенный инструмент FastAPI Foundry для извлечения чистого текста из файлов и веб-страниц перед их индексацией в RAG систему.

    Поддерживаемые форматы

    Категория Форматы
    Документы PDF, DOCX, DOC, ODT, RTF
    Таблицы XLSX, XLS, CSV, ODS
    Презентации PPTX, PPT
    Изображения (OCR) JPG, PNG, TIFF, BMP, GIF, WebP
    Веб HTML, HTM, Markdown
    Структурированные данные JSON, XML, YAML
    Исходный код Python, JS, TS, PHP, Go, Rust, C/C++, C#, Java, Shell, PowerShell и др.
    Архивы ZIP, RAR, 7Z, TAR, GZ (рекурсивно)
    Email EML, MSG
    Книги EPUB

    Автоизвлечение при индексации RAG

    Да — при построении RAG индекса через POST /api/v1/rag/build или кнопку Build Index в веб-интерфейсе Text Extractor запускается автоматически для каждого файла в указанной директории.

    Процесс: 1. RAGIndexer рекурсивно обходит директорию. 2. Для каждого файла вызывается TextExtractor — тот же движок, что используется в ручном режиме. 3. Извлечённый текст нарезается на чанки заданного размера с перекрытием. 4. Чанки векторизуются и сохраняются в FAISS индекс.

    Таким образом, если в папке лежат PDF, DOCX, изображения с текстом (OCR) и архивы — все они будут обработаны автоматически без ручного извлечения.

    ⚠️ Для OCR изображений необходим установленный Tesseract. Без него изображения будут пропущены без ошибки.

    Откройте вкладку RAG → прокрутите вниз до секции Извлечение текста.

    Извлечение из файла

    1. Перейдите на вкладку Файл в секции Text Extractor.
    2. Нажмите на поле выбора файла и выберите нужный документ.
    3. Нажмите кнопку Извлечь.
    4. Результат появится в текстовом поле ниже.
    5. Используйте кнопку Копировать для копирования текста в буфер обмена.

    Извлечение из URL

    1. Перейдите на вкладку URL в секции Text Extractor.
    2. Введите адрес страницы или прямую ссылку на файл.
    3. Опционально включите:
    4. JavaScript — для страниц с динамическим контентом (требует Playwright)
    5. OCR изображений — для распознавания текста на картинках страницы
    6. Нажмите Извлечь.

    ⚠️ Извлечение с URL с включённым JavaScript занимает больше времени (до 30 секунд).

    Использование через API

    Извлечение из файла

    POST /api/v1/rag/extract/file
Content-Type: multipart/form-data

file: <binary>

    Ответ: 

    {
  "success": true,
  "filename": "document.pdf",
  "count": 1,
  "total_chars": 15420,
  "files": [
    {
      "filename": "document.pdf",
      "path": "document.pdf",
      "size": 204800,
      "type": "pdf",
      "text": "Извлечённый текст..."
    }
  ]
}

    Извлечение из URL

    POST /api/v1/rag/extract/url
Content-Type: application/json

{
  "url": "https://example.com/article",
  "enable_javascript": false,
  "process_images": false,
  "web_page_timeout": 30
}

    Список поддерживаемых форматов

    GET /api/v1/rag/extract/formats

    Работа с архивами

    При загрузке архива (ZIP, RAR, 7Z, TAR) Text Extractor:

    1. Распаковывает архив во временную директорию.
    2. Рекурсивно обрабатывает все поддерживаемые файлы внутри.
    3. Возвращает массив результатов — по одному элементу на каждый файл.
    4. Поддерживает вложенные архивы (до 3 уровней).

    Ограничения архивов: - Максимальный размер архива: 20 МБ - Максимальный размер распакованного содержимого: 100 МБ - Максимальная глубина вложенности: 3

    OCR (распознавание текста на изображениях)

    Для работы OCR необходим установленный Tesseract:

    # Windows (через Chocolatey)
choco install tesseract

# Или скачать установщик с https://github.com/UB-Mannheim/tesseract/wiki

    По умолчанию распознаются русский и английский языки (rus+eng). Изменить языки можно через переменную окружения:

    OCR_LANGUAGES=rus+eng+deu

    Ограничения

    Настройки управляются через вкладку Settings → Text Extractor в веб-интерфейсе или напрямую в config.json секция text_extractor.

    Параметр По умолчанию Описание
    Макс. размер файла 20 МБ Максимальный размер загружаемого файла
    Таймаут обработки 300 секунд Максимальное время обработки одного файла
    Языки OCR rus+eng Языки Tesseract через +, например rus+eng+deu
    Таймаут веб-запроса 30 секунд Таймаут загрузки веб-страницы
    Макс. изображений/страница 20 Максимум изображений для OCR на одной странице
    JavaScript выключен JS-рендеринг через Playwright
    Ограничения ресурсов включено Лимиты памяти для LibreOffice и Tesseract (Unix)

    Безопасность

    • URL-запросы проверяются на SSRF: заблокированы localhost, внутренние IP-диапазоны и Docker-сети.
    • MIME-тип файла проверяется на соответствие расширению.
    • Имена файлов из архивов санитизируются (защита от path traversal).
