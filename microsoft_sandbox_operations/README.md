# 🖥️ Операции с Windows Sandbox

Директория содержит набор инструментов для оркестрации, подготовки и запуска проекта в изолированной среде **Windows Sandbox**. Эти инструменты обеспечивают безопасное тестирование кода, исключая влияние на хост-систему.

## 📋 Обзор инструментов

Инфраструктура состоит из трех основных компонентов, работающих в связке:

| Файл | Тип | Описание |
| :--- | :--- | :--- |
| `sandbox-launcher.ps1` | PowerShell | Главный оркестратор (лончер) всего процесса. |
| `sandbox-mapper.ps1` | PowerShell | Утилита синхронизации исходного кода в staging-директорию. |
| `sandbox.wsb` | XML / WSB | Конфигурационный файл параметров песочницы. |

---

## ⚙️ Описание компонентов

### 1. sandbox-launcher.ps1
**Назначение:** Инициализация и контроль жизненного цикла песочницы.

**Логика выполнения:**
- **Проверка окружения:** Поиск необходимых скриптов и конфигураций.
- **Вызов синхронизации:** Запуск `sandbox-mapper.ps1` для подготовки актуальной копии кода.
- **Стабилизация:** Ожидание завершения дисковых операций перед монтированием.
- **Запуск процесса:** Инициализация Windows Sandbox через `.wsb` файл.

### 2. sandbox-mapper.ps1
**Назначение:** Создание идентичной копии проекта в промежуточной папке (staging) для последующего монтирования.

**Технические особенности:**
- **Использование Robocopy:** Применение режима зеркалирования (`/MIR`) для максимальной производительности.
- **Фильтрация артефактов:** Автоматическое исключение директорий `.git`, `venv`, кешей Python (`__pycache__`), логов и бинарных файлов сборки.
- **Инкрементальное обновление:** Копируются только измененные файлы, что ускоряет повторные запуски.

### 3. sandbox.wsb
**Назначение:** Декларативное описание параметров виртуальной машины Sandbox.

**Конфигурация включает:**
- **Mapped Folders:** Связывание staging-директории хоста с внутренним путем в песочнице (обычно `C:\Users\WDAGUtilityAccount\Desktop\Sandbox`).
- **Networking:** Настройка сетевого доступа.
- **Logon Command:** Автоматическое выполнение скриптов подготовки внутри песочницы при старте.

## 🚀 Использование

Для запуска среды выполните команду в консоли PowerShell от имени администратора:

```powershell
# Переход в директорию операций
cd .\microsoft_sandbox_operations\

# Запуск оркестратора
powershell -ExecutionPolicy Bypass -File .\sandbox-launcher.ps1
```

### Параметры запуска

`sandbox-launcher.ps1` поддерживает следующие аргументы:
- `-DelayMs`: Время задержки в миллисекундах между синхронизацией и запуском (по умолчанию `2000`).

`sandbox-mapper.ps1` поддерживает следующие аргументы:
- `-Source`: Путь к исходному коду проекта.
- `-Target`: Путь к целевой папке для staging.

## ⚠️ Требования

1. **ОС:** Windows 10/11 Pro, Enterprise или Education.
2. **Компонент:** Включенная функция "Windows Sandbox" (Песочница Windows).
3. **Права:** Права администратора для корректной работы `robocopy` и монтирования дисков.
