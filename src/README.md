# 📂 Исходный код (src)

Эта директория содержит всю основную логику приложения FastAPI Foundry.

---
**📚 Документация:** [Главное README](../README.md) | [Руководство для разработчиков](../docs/development.md) | [Гайд по логированию](../docs/logging_guidelines.md)
---

## 🏗️ Архитектура

FastApiFoundry имеет модульную структуру для лучшей организации и масштабируемости.

| Директория | Описание |
| :--- | :--- |
| **`api/`** | Содержит веб-приложение **FastAPI**. Здесь находятся все эндпоинты (`endpoints`), middleware и основная конфигурация приложения (`app.py`, `main.py`). |
| **`core/`** | Ядро приложения. Включает в себя загрузку и управление конфигурацией из `config.json` и `.env` файлов. |
| **`models/`** | Логика для взаимодействия с AI-моделями. Включает клиент для Foundry (`foundry_client.py`) и менеджер моделей (`model_manager.py`) для управления несколькими провайдерами. |
| **`rag/`** | Реализация **Retrieval-Augmented Generation (RAG)**. Отвечает за создание, загрузку и поиск по векторному индексу документов (`rag_system.py`). |
| **`utils/`** | Вспомогательные утилиты. Ключевым является продвинутая **система логирования** (`logging_system.py`), которая поддерживает структурированные JSON-логи, измерение производительности и ротацию файлов. |
| `logger/` | Устаревшая система логирования. Сохранена для обратной совместимости. Новым модулям следует использовать `get_logger` из `src.utils.logging_system`. |

## 🚀 Ключевые файлы

- `src/api/app.py`: "Фабрика" для создания и конфигурации экземпляра FastAPI.
- `src/api/main.py`: Точка входа для запуска сервера `uvicorn`.
- `src/core/config.py`: Загружает и предоставляет доступ к настройкам приложения.
- `src/models/model_manager.py`: Управляет жизненным циклом и вызовами AI-моделей.
- `src/rag/rag_system.py`: Инкапсулирует логику RAG-индекса и поиска.
- `src/utils/logging_system.py`: Основная система логирования.

## 👨‍💻 Для разработчиков

- **Добавление эндпоинтов**: Создайте новый файл в `src/api/endpoints/` и подключите его в `src/api/app.py`. Подробнее см. в [руководстве по разработке](../../docs/development.md).
- **Использование логгера**: Получите экземпляр логгера через `from src.utils.logging_system import get_logger`. Подробнее в [гайдлайнах по логированию](../../docs/logging_guidelines.md).
- **Добавление зависимостей**: Добавьте пакет в `requirements.txt` и пересоберите Docker-образ, если это необходимо.
