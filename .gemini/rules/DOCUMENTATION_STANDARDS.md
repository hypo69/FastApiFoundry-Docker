# Стандарты документации проекта FastAPI Foundry

Этот файл определяет правила создания и обновления документации для ИИ-агентов и разработчиков.

## 1. Структура разделов

Документация должна быть разделена на два основных блока:

### Руководство пользователя (User Guide)
- **Целевая аудитория:** Конечные пользователи, администраторы.
- **Язык:** Простой, без глубоких технических деталей реализации.
- **Обязательные темы:** Установка (EXE/PS1), запуск, использование веб-интерфейса, настройка моделей, работа с чатом и RAG.
- **Локация:** `docs/ru/user/`

### Руководство разработчика (Developer Guide)
- **Целевая аудитория:** Программисты, системные архитекторы.
- **Язык:** Технический, с описанием API, структуры классов, логики взаимодействия компонентов.
- **Обязательные темы:** Архитектура (Foundry + FastAPI), описание эндпоинтов, расширение системы (агенты, трансляторы), работа с векторами (FAISS).
- **Локация:** `docs/ru/dev/`

## 2. Форматирование
- Использовать GitHub Flavored Markdown.
- Для кода — соответствующие блоки с указанием языка (python, powershell, json).
- Обязательно наличие оглавления в больших файлах.
- Ссылки между документами должны быть относительными.

## 3. Процесс генерации
1. **Анализ файла:** Понять роль файла в системе.
2. **Логирование:** Записать путь к файлу в `docs/ru/processed_files.log`.
3. **Обновление индекса:** Добавить ссылку на новый документ в `docs/ru/INDEX.md`.

## 4. Как пользоваться этим правилом
При создании документации всегда сверяйтесь с этим файлом. Если вы добавляете новую функциональность, вы обязаны обновить соответствующие разделы в `user/` и `dev/` директориях.
