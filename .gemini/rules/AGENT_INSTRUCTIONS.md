## 🔤 КОДИРОВКА ФАЙЛОВ — АБСОЛЮТНОЕ ПРАВИЛО

**ВСЕ файлы проекта ТОЛЬКО UTF-8 без BOM.**

### При чтении/записи файлов через инструменты:

- `executeBash` + PowerShell → **ВСЕГДА** использовать Python вместо PowerShell для работы с текстовыми файлами, потому что PowerShell на Windows использует CP1251 по умолчанию
- Если PowerShell неизбежен → явно указывать `-Encoding UTF8` для `Get-Content`/`Set-Content`/`Out-File`
- **ПРЕДПОЧТИТЕЛЬНЫЙ инструмент для чтения/записи .md, .py, .js, .json, .yaml, .ps1** — `fsRead` / `fsWrite` / `fsReplace` (они всегда UTF-8)
- При использовании Python для файловых операций: `open(path, encoding='utf-8')`, при записи — `open(path, 'w', encoding='utf-8')`
- BOM (`\xef\xbb\xbf`) — **удалять** при обнаружении, никогда не добавлять

```
❌ ЗАПРЕЩЕНО: PowerShell Get-Content без -Encoding UTF8
❌ ЗАПРЕЩЕНО: записывать файлы с BOM
✅ ПРАВИЛЬНО: fsRead / fsWrite / fsReplace для всех текстовых файлов
✅ ПРАВИЛЬНО: Python open(path, encoding='utf-8') если нужен скрипт
✅ ПРАВИЛЬНО: PowerShell только с явным -Encoding UTF8
```
