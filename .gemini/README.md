# .gemini — agent instruction memory

Эта директория предназначена для AI-агентов, работающих с проектом.

## Назначение

`.gemini` хранит общие правила поведения, проектную память, технические стандарты и рабочие инструкции для агента. Эти правила изменчивы: пользователь может корректировать их по ходу разработки, а агент должен воспринимать такие корректировки как актуальные команды для будущей работы.

## Обязательное правило запуска

При каждом новом запуске среды разработки или агентской сессии агент обязан прочитать `.gemini` перед работой с проектом. Это относится к VS Code, LM Studio, PyCharm, терминальным сессиям, локальным агентам и любым другим средам разработки.

Минимальный порядок чтения:

1. `.gemini/README.md`
2. `.gemini/rules/AGENT_INSTRUCTIONS.md`
3. `.gemini/rules/AMAZON_MD.md`
4. `.gemini/rules/CODE_RULES.md`
5. `.gemini/rules/POWERSHELL_CODE_RULES.md`
6. `.gemini/rules/memory-bank/guidelines.md`
7. `.gemini/rules/memory-bank/knowledge-base.md`
8. Остальные файлы `.gemini/rules/` и `.gemini/rules/memory-bank/` по задаче

## Правило изменения

`.gemini` не является защищенной или запретной зоной. Агент может изменять эти файлы, когда пользователь дает новые правила, корректирует поведение агента или просит обновить память проекта.

## Обзор файлов

- `rules/AGENT_INSTRUCTIONS.md` — главные поведенческие инструкции агента.
- `rules/AMAZON_MD.md` — сводная конфигурация агента и архитектурные правила проекта.
- `rules/CODE_RULES.md` — стандарты кода и документирования.
- `rules/CODE_REVISION.md` — чеклист ревизии кода.
- `rules/DEPLOYMENT-WORKFLOW.md` — правила деплоя и проверок.
- `rules/POWERSHELL_CODE_RULES.md` — стиль PowerShell-кода.
- `rules/sync_doc.md` — синхронизация документации после изменений.
- `rules/VERSION.md` — правила версионирования.
- `rules/memory-bank/` — проектная память: продукт, структура, стек, база знаний и практические руководства.
