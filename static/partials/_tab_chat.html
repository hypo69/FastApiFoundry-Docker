Как работают модели в чате
  • Список: показываются модели из локальных кэшей: Foundry берём из /foundry/models/cached, HuggingFace — из /hf/models (скачанные), llama.cpp — из /llama/models (локальные .gguf в ~/.models).
  • Переключение: при выборе модели выгружаются активные модели из памяти (Foundry unload + HF unload) и при необходимости останавливается llama.cpp, затем загружается выбранная и UI ждёт готовности (Foundry: /foundry/models/loaded, HF: /hf/models, llama.cpp: /llama/status как running).
  • Как добавить через интерфейс: для Foundry — вкладка Foundry, для HuggingFace — вкладка HuggingFace. Для llama.cpp — скопируйте .gguf в ~/.models во вкладке llama.cpp, затем Refresh — модель появится в списке.
Загрузка через CLI (в обход интерфейса)
  • Foundry: foundry model download deepseek-r1-distill-qwen-7b-generic-cpu:3 foundry model load deepseek-r1-distill-qwen-7b-generic-cpu:3
  • HuggingFace CLI: pip install huggingface_hub hf auth login hf download bartowski/gemma-2-2b-it-GGUF --local-dir ~/.models
  • llama.cpp сервер: llama-server --model ~/.models/gemma-2-2b-it-Q6_K.gguf --port 9780 Или через скрипт: .\scripts\llama-start.ps1 -ModelPath ~/.models/gemma-2-2b-it-Q6_K.gguf
Получить HuggingFace модель через интерфейс
  1. Откройте вкладку HuggingFace.
  2. Если модель gated (требует принятия лицензии) — задайте токен в Settings → HuggingFace и примите лицензию на странице модели.
  3. В блоке Download Model введите model_id (например google/gemma-2b) и нажмите Download.
  4. Нажмите Refresh в Downloaded Locally — модель появится в списке чата.
