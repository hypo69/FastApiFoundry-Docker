How models work in chat
  • Models: loaded from local caches: Foundry via /foundry/models/cached, HuggingFace — via /hf/models (downloaded), llama.cpp — via /llama/models (local .gguf files in ~/.models).
  • Switching: selecting a model unloads active models from memory (Foundry unload + HF unload) and stops llama.cpp if needed, then loads the selected model and waits for readiness (Foundry: /foundry/models/loaded, HF: /hf/models, llama.cpp: /llama/status = running).
  • To add via the UI: for Foundry open the Foundry tab, for HuggingFace open HuggingFace. For llama.cpp copy a .gguf to ~/.models and open llama.cpp, then Refresh — the model will appear in the list.
Load via CLI (bypassing the UI)
  • Foundry: foundry model download deepseek-r1-distill-qwen-7b-generic-cpu:3 foundry model load deepseek-r1-distill-qwen-7b-generic-cpu:3
  • HuggingFace CLI: pip install huggingface_hub hf auth login hf download bartowski/gemma-2-2b-it-GGUF --local-dir ~/.models
  • llama.cpp server: llama-server --model ~/.models/gemma-2-2b-it-Q6_K.gguf --port 9780 Or via script: .\scripts\llama-start.ps1 -ModelPath ~/.models/gemma-2-2b-it-Q6_K.gguf
Download a HuggingFace model via the web UI
  1. Open the HuggingFace tab.
  2. If the model is gated (requires license acceptance), set your token in Settings → HuggingFace and accept the license on the model page.
  3. In Download Model enter the model_id (e.g. google/gemma-2b) and click Download.
  4. Click Refresh in Downloaded Locally — the model will appear in the chat list.

How to download a model via Foundry CLI
  1. Open PowerShell or Terminal.
  2. List available models: foundry model list
  3. Download a model: foundry model download <model-id>
  4. Load the model: foundry model load <model-id>
Example: foundry model download qwen2.5-0.5b-instruct-generic-cpu:4, then foundry model load qwen2.5-0.5b-instruct-generic-cpu:4
