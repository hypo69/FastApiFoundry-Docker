# 🌐 Веб-интерфейс FastAPI Foundry

Веб-консоль для управления AI моделями, чата и мониторинга системы.

**Адрес после запуска:** http://localhost:9696

---

## 📁 Файлы

| Файл | Описание |
|------|----------|
| `index.html` | Главная страница — все вкладки интерфейса |
| `app.js` | Логика интерфейса (API запросы, обновление UI) |
| `chat.html` | Отдельная страница чата |
| `ai.html` | Отдельная страница управления моделями |
| `control.html` | Отдельная панель управления системой |
| `simple.html` | Упрощённый интерфейс |
| `available_models.json` | Список моделей для выпадающего списка |

---

## 🗂️ Вкладки главной страницы

Главная страница (`index.html`) содержит 8 вкладок:

### 💬 Chat — Чат с AI

Основная вкладка для общения с моделью.

- Поле ввода сообщения, отправка по Enter или кнопке Send
- Выбор модели из списка (или Auto-select — первая доступная)
- Регулировка Temperature (0–2) — чем выше, тем более случайные ответы
- Max Tokens — максимальная длина ответа
- Чекбокс "Use RAG Context" — использовать ли поиск по документации при ответе
- Выбранная модель автоматически сохраняется как модель по умолчанию в `config.json`

### 🤖 Models — Подключённые модели

Список моделей, которые сейчас доступны через Foundry.

- Карточка каждой модели: ID, провайдер, статус, максимум токенов
- Кнопка "Use in Chat" — переключает на вкладку Chat с этой моделью
- Кнопка "Test" — отправляет тестовый запрос "Hello!" и показывает ответ
- Кнопка "Remove" — выгружает модель из памяти
- Radio-кнопка "Use as default model" — сохраняет выбор в `config.json`
- Кнопка "Add Model" — форма добавления модели (Foundry, Ollama, OpenAI, Anthropic, Custom)

### ⬇️ Foundry — Управление Foundry

Управление сервисом Microsoft Foundry Local.

- Кнопки Start / Stop Foundry
- Индикатор статуса и порта (порт определяется автоматически)
- Лог вывода Foundry в реальном времени
- Список загруженных моделей с кнопками "Использовать" и удалить
- Выпадающий список для загрузки новой модели + прогресс-бар загрузки
- Прогресс проверяется каждые 5 секунд, таймаут — 5 минут

### 🔍 RAG — Система поиска по документации

RAG (Retrieval-Augmented Generation) — поиск релевантного контекста в документах перед ответом AI.

- Статус системы: включена / выключена, количество проиндексированных фрагментов
- Настройки: директория индекса, размер фрагмента, модель эмбеддингов, Top K результатов
- Кнопка "Rebuild Index" — переиндексировать документы
- Кнопка "Clear All Chunks" — удалить весь индекс (с подтверждением)
- Кнопка "Test Search" — проверить поиск по введённому запросу, показывает результаты с оценкой релевантности

### ⚙️ Settings — Настройки

Редактирование `config.json` через веб-форму.

Секции настроек:
- **FastAPI Server** — хост, порт, режим (dev/prod), количество воркеров, автопоиск свободного порта
- **Foundry AI** — URL бэкенда, модель по умолчанию, temperature, max tokens, автозагрузка модели при старте
- **RAG System** — включить/выключить, директория индекса, размер чанка; кнопка очистки RAG
- **Security** — API ключ, включить HTTPS
- **Logging** — уровень логов (DEBUG/INFO/WARNING/ERROR), путь к файлу лога
- **Development** — режим отладки, подробное логирование

Кнопки "Reload" и "Save All" — загрузить текущие значения из файла / сохранить изменения.

### ▶️ Examples — Примеры SDK

Запуск встроенных демонстрационных примеров прямо из браузера.

| Кнопка | Что делает |
|--------|-----------|
| Basic SDK Usage | Базовый пример генерации текста через SDK |
| RAG Search Demo | Поиск через RAG систему |
| Batch Generation | Пакетная генерация нескольких запросов |
| Models & Config | Получение списка моделей и конфигурации |

Результат выполнения отображается в консольном окне справа.

### 📋 Logs — Системные логи

Просмотр логов сервера.

- Загружаются автоматически при переходе на вкладку
- Фильтр по уровню: All / Debug / Info / Warning / Error
- Кнопки обновить и очистить вид
- Боковая панель: общий статус системы (healthy / warning / critical), сводка ошибок и предупреждений, количество записей

### 📖 API Documentation

Встроенная Swagger UI документация (`/docs`) в iframe.

- Кнопка "Open in New Tab" — открыть в отдельной вкладке
- Интерактивное тестирование всех API endpoints прямо из браузера

## 🔄 Автоматическое поведение

При загрузке страницы интерфейс автоматически:

1. Загружает конфигурацию из `/api/v1/config`
2. Проверяет статус системы (`/api/v1/health`) — обновляет индикатор в шапке
3. Загружает список доступных моделей и заполняет селектор в чате
4. Проверяет доступность модели по умолчанию из `config.json`
5. Повторяет проверку статуса каждые 30 секунд

Индикатор в шапке:
- 🟢 **Connected** — API и Foundry работают
- 🟡 **API Only** — API работает, Foundry недоступен
- 🔴 **Error / Offline** — сервер недоступен
