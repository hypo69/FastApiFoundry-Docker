# scripts/start/ — Архив скриптов запуска

Директория содержит **устаревшие итерации** скрипта запуска FastAPI Foundry.
Все файлы здесь — промежуточные версии, созданные в процессе отладки.

**Актуальный скрипт запуска:** `../../start.ps1` (в корне проекта)

---

## Файлы

### start_working.ps1
Версия с исправленным поиском Foundry через `tasklist` + `netstat`.
Ищет порт сначала на фиксированном `50477`, затем через процесс `foundry`.
Отличие от `start.ps1`: использует `python311.exe` вместо `python.exe`.

### start_fixed_foundry.ps1
Версия с пошаговой проверкой Foundry: CLI → статус сервиса → API на порту.
Сканирует диапазон портов `50000–60000` если `50477` не отвечает.
Отличие: явный вызов `foundry service status` перед запуском.

### start_verbose.ps1
Версия с максимальным выводом диагностики.
Передаёт весь stdout Foundry в консоль (`=== FOUNDRY SERVICE STATUS ===`).
Использовалась для отладки проблем с определением порта.

### start_corrected.ps1
Версия с исправленными регулярными выражениями для PowerShell.
Упрощён regex для парсинга порта из `netstat` и вывода Foundry.
Использует `python.exe` (не `python311.exe`).

### start_new.ps1
Промежуточная версия — идентична `start.ps1` по логике,
но ищет порт только через `Inference.Service.Agent*` процесс без fallback.

### start_fixed.ps1
Ранняя версия с поиском через `Get-Process -Name "foundry"`.
Парсит порт из вывода `foundry service start` по паттерну `http://127.0.0.1:PORT/`.
Не работает с `Inference.Service.Agent` — устарела.

### start_minimal.ps1
Минимальная версия без автозапуска Foundry.
Проверяет только фиксированный порт `50477`, не ищет динамический.
Использовалась для быстрой проверки когда Foundry уже запущен.

### start_utf8.ps1
Версия с принудительной установкой UTF-8 кодировки для консоли.
Добавляет `$OutputEncoding`, `[Console]::OutputEncoding`, `PYTHONIOENCODING=utf-8`.
Решала проблему с кириллицей в выводе на Windows.
Логика запуска — аналог `start_minimal.ps1`.

---

## Почему не удалены

Сохранены как история решений конкретных проблем:
- проблема с кодировкой → `start_utf8.ps1`
- проблема с определением порта → `start_verbose.ps1`, `start_fixed_foundry.ps1`
- проблема с именем процесса → `start_fixed.ps1` vs `start_working.ps1`
